# 06-02 动态菜单 — 编码规范（Spec）

## 1. 接口规范

### 1.1 获取当前用户可访问菜单

```
GET /menus/user
Authorization: Bearer {accessToken}
```

| 参数 | 必填 | 说明                                   |

## 2. 后端权限过滤逻辑

### 2.1 用户菜单查询流程

```
GET /menus/user
  │
  ├─ 1. 从 Token 解析 userId
  ├─ 2. 判断是否为超级管理员
  │      └─ 是 → 返回全部门户菜单
  ├─ 3. 查询缓存 menu:user:{userId}:portal
  │      └─ 命中 → 直接返回
  ├─ 4. 从 DB 计算
  │      ├─ 查询 sys_user_role → 用户角色 ID 列表（角色状态=正常）
  │      ├─ 查询 sys_role_menu → 角色关联的菜单 ID 列表（合并去重）
  │      ├─ 查询 sys_menu → 按条件过滤
  │      │      ├─ id IN (角色关联的菜单 ID)
  │      │      ├─ status = 1
  │      │      ├─ menu_type IN ('M', 'C')（不返回按钮）
  │      │      └─ app_id IS NULL
  │      ├─ 构建树结构
  │      └─ 过滤空目录（目录下无子节点的移除）
  ├─ 5. 写入缓存 menu:user:{userId}:portal
  └─ 6. 返回
```

### 2.2 子应用入口过滤

```
GET /apps/user
  │
  ├─ 1. 从 Token 解析 userId
  ├─ 2. 判断是否为超级管理员
  │      └─ 是 → 返回所有已启用的子应用
  ├─ 3. 查询缓存 apps:user:{userId}
  │      └─ 命中 → 直接返回
  ├─ 4. 从 DB 计算
  │      ├─ 查询用户通过角色关联的所有菜单 ID
  │      ├─ 从 sys_menu 中提取这些菜单对应的 app_id（去重、排除 NULL）
  │      ├─ 查询 sys_app 表获取应用详情（仅状态=正常的）
  │      └─ 按 sort_order 排序
  ├─ 5. 写入缓存 apps:user:{userId}
  └─ 6. 返回
```

### 2.3 空目录过滤算法

```java
// 伪代码：从叶子节点往上过滤，移除无可见子节点的目录
private List<MenuVO> filterEmptyDirs(List<MenuVO> tree) {
    Iterator<MenuVO> it = tree.iterator();
    while (it.hasNext()) {
        MenuVO node = it.next();
        if (node.getMenuType().equals("M")) {
            node.setChildren(filterEmptyDirs(node.getChildren()));
            if (node.getChildren().isEmpty()) {
                it.remove();
            }
        }
    }
    return tree;
}
```

## 3. 缓存设计

### 3.1 缓存 Key

| Key                          | 内容                       | TTL     |
| ---------------------------- | -------------------------- | ------- |
| `menu:user:{userId}:portal`  | 用户的门户菜单树           | 30 分钟 |
| `menu:user:{userId}:{appId}` | 用户在指定子应用的菜单树   | 30 分钟 |
| `apps:user:{userId}`         | 用户有权限的子应用入口数据 | 30 分钟 |

### 3.2 缓存刷新

| 触发事件 | 清除的缓存 |
| --- | --- |
| 角色功能权限变更 | 该角色所有用户的 `menu:user:*` 和 `apps:user:*` |
| 用户角色变更 | 该用户的 `menu:user:*` 和 `apps:user:*` |
| 菜单增删改 | 全部 `menu:user:*` 和 `apps:user:*`（或按 appId 精确清除） |
| 应用状态变更 | 全部 `apps:user:*` |

## 4. 前端两级加载实现

### 4.1 状态管理

```typescript
interface MenuState {
  portalMenus: MenuVO[];
  userApps: AppEntry[];
  currentAppId: string | null; // 当前所在应用，null=门户
}

const useMenuStore = defineStore('menu', {
  state: (): MenuState => ({
    portalMenus: [],
    userApps: [],
    currentAppId: null,
  }),

  getters: {
    // 当前应显示的侧边栏菜单
    sidebarMenus(state): MenuVO[] {
      return state.currentAppId ? [] : state.portalMenus;
    },
  },

  actions: {
    // 第一级：加载门户菜单 + 子应用入口数据
    async loadPortalMenus() {
      const [menuRes, appRes] = await Promise.all([
        api.get('/menus/user'),
        api.get('/apps/user'),
      ]);
      this.portalMenus = menuRes.data;
      this.userApps = appRes.data;
    },

    // 切换当前应用
    switchApp(appId: string | null) {
      this.currentAppId = appId;
    },

    // 清空（退出登录时调用）
    clear() {
      this.portalMenus = [];
      this.userApps = [];
      this.currentAppId = null;
    },
  },
});
```

### 4.2 菜单转路由

```typescript
function menusToRoutes(menus: MenuVO[]): RouteRecordRaw[] {
  return menus.map((menu) => {
    const route: RouteRecordRaw = {
      path: menu.path,
      name: menu.menuName,
      component: resolveComponent(menu.component),
      meta: {
        title: menu.menuName,
        icon: menu.icon,
        keepAlive: menu.isCache,
        visible: menu.visible,
      },
      children: menu.children?.length ? menusToRoutes(menu.children) : [],
    };
    return route;
  });
}

function resolveComponent(component: string | null) {
  if (!component) return () => import('@/layout/index.vue');
  return () => import(`@/views/${component}.vue`);
}
```

### 4.3 动态路由注册

```typescript
// 注册门户路由
function registerPortalRoutes(menus: MenuVO[]) {
  const routes = menusToRoutes(menus);
  routes.forEach((route) => router.addRoute(route));
}
```

## 5. 门户首页（欢迎页）

### 5.1 首页定位

```text
/dashboard 作为门户首页路由
  ├─ 仅展示欢迎文案与基础品牌信息
  ├─ 不承载应用卡片、待办、公告聚合等业务组件
  └─ 子应用切换通过应用导航栏、菜单搜索或直接路由进入
```

### 5.2 进入子应用逻辑

```typescript
async function enterApp(app: AppEntry) {
  const menuStore = useMenuStore();

  // 切换当前应用
  menuStore.switchApp(app.appId);

  // 跳转到子应用入口，具体挂载与内部路由由微前端接入层处理
  router.push(app.routePrefix);
}
```

## 6. 权限变更后菜单刷新

### 6.1 WebSocket 通知（可选）

```
后端权限变更 → WebSocket 推送 { type: "MENU_CHANGED", userId }
前端收到 → 清空 menuStore 中的缓存 → 重新加载当前应用的菜单
```

### 6.2 路由守卫兜底

```typescript
// 路由守卫中检查菜单是否需要刷新
router.beforeEach(async (to) => {
  const menuStore = useMenuStore();

  // 门户菜单未加载 → 加载
  if (!menuStore.portalMenus.length) {
    await menuStore.loadPortalMenus();
    registerPortalRoutes(menuStore.portalMenus);
  }

  // 识别子应用
  const app = menuStore.userApps.find((a) => to.path.startsWith(a.routePrefix));
  if (app) {
    menuStore.switchApp(app.appId);
  } else {
    menuStore.switchApp(null);
  }
});
```

## 8. 安全要点

| 项目           | 要求                                        |
| -------------- | ------------------------------------------- |
| 后端强制过滤   | 菜单接口后端按用户权限过滤，不依赖前端传参  |
| 应用入口校验   | `/apps/user` 仅返回用户有权访问的子应用入口 |
| 前端缓存清理   | 退出登录时清空所有菜单缓存                  |
| 不暴露无权菜单 | 无权限的菜单不在任何接口中返回              |
| 缓存一致性     | 权限变更后清除相关用户的菜单缓存            |
