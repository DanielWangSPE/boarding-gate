# 06-03 路由联动 — 编码规范（Spec）

## 1. 路由架构

### 1.1 路由分层

```
主应用路由表
  │
  ├─ 静态路由（无需权限）
  │      ├─ /login              登录页
  │      ├─ /403                无权限页
  │      ├─ /404                页面不存在
  │      └─ /redirect/:path     重定向中转
  │
  ├─ 门户动态路由（登录后注册）
  │      ├─ /dashboard          门户首页（欢迎页）
  │      ├─ /system/*           系统管理（用户、角色、菜单等）
  │      └─ /profile            个人中心
  │
  ├─ 子应用路由前缀占位（登录后按接入配置注册）
  │      ├─ /project/*          项目管理子应用
  │      ├─ /hr/*               人事管理子应用
  │      └─ ...                 其他子应用
  │
  └─ 兜底路由
         └─ /:pathMatch(.*)*   404
```

### 1.2 路由注册时序

```
阶段一：应用初始化
  └─ 注册静态路由（login, 403, 404, redirect）

阶段二：登录成功 / 页面刷新（路由守卫触发）
  ├─ 加载门户菜单 → 注册门户动态路由
  ├─ 加载子应用入口列表 → 注册子应用路由前缀通配
  │      每个子应用注册一条: { path: '/project/:pathMatch(.*)*', component: SubAppLoader }
  └─ 注册兜底 404 路由

阶段三：进入子应用（路由守卫触发）
  ├─ 识别子应用前缀 → 同步当前应用上下文
  └─ 激活微前端子应用
```

### 1.3 子应用路由前缀通配

```typescript
// 阶段二：为每个子应用注册通配路由
function registerAppPlaceholders(apps: AppEntry[]) {
  apps.forEach((app) => {
    router.addRoute({
      path: `${app.routePrefix}/:pathMatch(.*)*`,
      name: `app-placeholder-${app.appId}`,
      component: () => import('@/views/AppLoading.vue'),
      meta: {
        appId: app.appId,
        isAppPlaceholder: true,
      },
    });
  });
}

// 子应用具体挂载与内部路由由微前端接入层处理，此处仅保留前缀占位路由
```

## 2. 路由守卫

### 2.1 全局前置守卫

```typescript
const whiteList = ['/login', '/403', '/404'];

router.beforeEach(async (to, from, next) => {
  NProgress.start();

  const userStore = useUserStore();
  const menuStore = useMenuStore();

  // 1. 白名单直接放行
  if (whiteList.includes(to.path)) {
    return next();
  }

  // 2. 检查登录态
  if (!userStore.token) {
    return next({ path: '/login', query: { redirect: to.fullPath } });
  }

  // 3. 加载门户菜单（仅首次）
  if (!menuStore.portalMenus.length) {
    try {
      await menuStore.loadPortalMenus();
      registerPortalRoutes(menuStore.portalMenus);
      registerAppPlaceholders(menuStore.userApps);
      addCatchAllRoute();
      return next({ ...to, replace: true }); // 路由表变更，重新匹配
    } catch (err) {
      userStore.logout();
      return next({ path: '/login', query: { redirect: to.fullPath } });
    }
  }

  // 4. 检查是否命中子应用
  const app = menuStore.userApps.find((a) => to.path.startsWith(a.routePrefix));
  if (app) {
    menuStore.switchApp(app.appId);
  } else {
    menuStore.switchApp(null);
  }

  // 5. 权限校验（路由 meta 中无特殊权限标记则放行，具体按钮权限由组件控制）
  next();
});

router.afterEach(() => {
  NProgress.done();
});
```

### 2.2 登录后回跳

```typescript
// 登录页
async function handleLogin() {
  await userStore.login(loginForm);
  const redirect = route.query.redirect as string;
  router.push(redirect || '/dashboard');
}
```

**规则**：

- 登录页记录 `redirect` 查询参数
- 登录成功后跳转到 `redirect` 指向的路由
- 如果 `redirect` 为空或无效，跳转到门户首页 `/dashboard`（欢迎页）
- `redirect` 参数需做安全校验（防止开放重定向，仅允许同源路径）

### 2.3 子应用路由转发

```
子应用路由转发机制（微前端集成）
  │
  ├─ 主应用路由匹配到子应用前缀 → 渲染 SubAppLayout 组件
  ├─ SubAppLayout 负责：
  │      ├─ 挂载/激活子应用容器
  │      ├─ 将 Token、权限数据通过 props 传递给子应用
  │      └─ 将子路由路径传递给子应用
  ├─ 子应用内部使用自己的路由实例管理内部路由
  └─ 子应用路由变更时通知主应用更新 URL（保持 URL 同步）
```

## 3. 子应用容器组件

### 3.1 SubAppLayout

```vue
<template>
  <div class="sub-app-layout">
    <div :id="containerId" class="sub-app-container" />
  </div>
</template>

<script setup lang="ts">
import { onMounted, onUnmounted, watch } from 'vue';
import { useRoute } from 'vue-router';
import { useMenuStore } from '@/stores/menu';
import { loadMicroApp } from '@/micro/loader';

const route = useRoute();
const menuStore = useMenuStore();
const props = defineProps<{ appId: string }>();
const containerId = `sub-app-${props.appId}`;

let microApp: any = null;

onMounted(async () => {
  const app = menuStore.userApps.find((a) => a.appId === props.appId);
  if (!app) return;

  microApp = await loadMicroApp({
    name: app.appId,
    entry: app.entryUrl,
    container: `#${containerId}`,
    props: {
      routerBase: app.routePrefix,
      token: useUserStore().token,
      permissions: usePermissionStore().permissions,
    },
  });
});

// URL 变化时通知子应用
watch(
  () => route.fullPath,
  (newPath) => {
    microApp?.update?.({ currentPath: newPath });
  },
);

onUnmounted(() => {
  microApp?.unmount?.();
});
</script>
```

## 5. 路由元数据（meta）

```typescript
interface RouteMeta {
  title?: string; // 页面标题（面包屑、标签页显示）
  icon?: string; // 菜单图标
  keepAlive?: boolean; // 是否缓存
  visible?: boolean; // 是否在侧边栏显示
  appId?: string; // 所属子应用 ID
  appName?: string; // 所属子应用名称
  isAppPlaceholder?: boolean; // 子应用占位路由标记
  activeMenu?: string; // 高亮的菜单路径（用于详情页等隐藏路由）
}
```

## 6. URL 同步策略

```
主应用与子应用 URL 同步
  │
  ├─ 主应用 → 子应用
  │      ├─ 主应用路由变更时，通过 props 或事件通知子应用当前路径
  │      └─ 子应用根据 routerBase 截取相对路径，匹配内部路由
  │
  └─ 子应用 → 主应用
         ├─ 子应用路由变更时，通过 CustomEvent 通知主应用
         │      window.dispatchEvent(new CustomEvent('portal:route-change', {
         │        detail: { path: '/project/list/123' }
         │      }))
         ├─ 主应用监听事件，使用 router.replace 更新 URL
         └─ 保证浏览器地址栏始终反映真实路径
```

## 7. 编码规范

### 7.1 路由命名规范

| 规范       | 说明                                        |
| ---------- | ------------------------------------------- |
| name 唯一  | 每个路由的 name 必须全局唯一                |
| 子应用前缀 | 子应用路由 name 以 `app-{appId}-` 为前缀    |
| kebab-case | 路由 path 使用 kebab-case                   |
| 懒加载     | 所有页面组件使用 `() => import(...)` 懒加载 |

### 7.2 路由守卫注意事项

| 注意项 | 说明 |
| --- | --- |
| 避免死循环 | 路由守卫中 `next({ ...to, replace: true })` 需确保下次不再触发加载 |
| 错误处理 | 菜单加载失败需有降级方案（跳 403 或登录页） |
| 异步加载 | 菜单加载是异步的，需在加载完成后再 next() |
| 性能 | 路由守卫每次导航都执行，保持轻量（有缓存时 O(1) 判断） |

## 8. 错误码

| 错误码 | 说明                |
| ------ | ------------------- |
| 401    | 未登录或 Token 过期 |
| 403    | 已登录但无权访问    |
| 404    | 路由不存在          |

## 9. 安全要点

| 项目       | 要求                                              |
| ---------- | ------------------------------------------------- |
| 重定向校验 | `redirect` 参数仅允许同源相对路径，防止开放重定向 |
| 路由鉴权   | 前端路由守卫 + 后端接口鉴权双重保障               |
| 子应用隔离 | 子应用路由前缀不可重叠                            |
| Token 时效 | 路由守卫中检测 Token 有效性，过期则跳转登录       |
| 不可猜测   | 动态路由不预先注册，无权限的路由在路由表中不存在  |
