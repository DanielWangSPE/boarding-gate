# 06-01 菜单管理

## 1. 功能说明

本模块负责门户平台菜单资源的定义与维护，包括门户菜单信息管理和导航树的增删改查操作。子应用接入规则由微前端接入机制维护，本模块仅消费已接入应用信息用于门户侧展示与权限控制。

## 2. 功能清单

### 2.1 菜单信息

| 字段 | 说明 |
| --- | --- |
| 菜单名称 | 显示在侧边栏和面包屑中的名称 |
| 菜单图标 | 支持选择内置图标库或上传自定义图标 |
| 菜单类型 | 目录（M）/ 菜单（C）/ 按钮（F） |
| 路由路径 | 前端路由地址（目录和菜单类型必填） |
| 组件路径 | 门户页面对应的前端组件加载路径；子应用入口不在此处维护内部页面组件 |
| 权限标识 | 关联权限管理模块，格式如 `user:create`（按钮类型必填） |
| 所属应用 | 主应用（NULL）/ 子应用标识（app_id） |
| 父级菜单 | 上级菜单 ID，0 表示顶级 |
| 排序号 | 同级菜单的排列顺序 |
| 是否可见 | 控制菜单在侧边栏中的显隐，隐藏的菜单路由仍可访问 |
| 是否缓存 | 页面 keep-alive，切换后保留状态 |
| 是否外链 | 外部链接在新窗口打开 |
| 菜单状态 | 正常 / 停用 |
| 路由参数 | 默认携带的查询参数 |

### 2.2 菜单树操作

- 以树形结构管理所有菜单
- 支持新增、编辑、删除菜单节点
- 支持拖拽调整菜单顺序和层级
- 支持按应用维度筛选菜单树
- 支持菜单的快速启用/停用

## 3. 业务规则

| 编号 | 规则 | 说明 |
| --- | --- | --- |
| BR-01 | 菜单层级 | 最多 4 级（目录 > 目录 > 菜单 > 按钮） |
| BR-02 | 类型约束 | 按钮（F）不能有子节点；目录（M）下可挂目录、菜单；菜单（C）下只能挂按钮 |
| BR-03 | 名称唯一 | 同级菜单下名称不可重复 |
| BR-04 | 路由唯一 | 同一应用下路由路径不可重复 |
| BR-05 | 外链处理 | 外链类型菜单在新浏览器标签页中打开 |
| BR-06 | 删除限制 | 存在子菜单的菜单不可直接删除 |
| BR-07 | 权限标识唯一 | perms 字段全局唯一 |

## 4. 接口要求

| 接口             | 方法   | 说明                                      |
| ---------------- | ------ | ----------------------------------------- |
| /menus           | GET    | 获取完整菜单树（管理员，支持 appId 筛选） |
| /menus           | POST   | 创建菜单                                  |
| /menus/{id}      | PUT    | 更新菜单                                  |
| /menus/{id}      | DELETE | 删除菜单                                  |
| /menus/{id}/sort | PUT    | 调整菜单排序（拖拽）                      |

## 5. 页面清单

| 页面         | 说明                                                     |
| ------------ | -------------------------------------------------------- |
| 菜单管理页   | 树形结构管理菜单，支持增删改查、拖拽排序，支持按应用筛选 |
| 菜单编辑弹窗 | 新增/编辑菜单的表单弹窗                                  |
