# 06-01 菜单管理 — 编码规范（Spec）

## 1. 数据模型

### 1.1 菜单表（sys_menu）

> `sys_menu` 同时承载权限资源的定义（参见 05-01 功能权限 spec），此处侧重菜单管理视角的字段说明。

```sql
CREATE TABLE sys_menu (
  id            BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
  parent_id     BIGINT       NOT NULL DEFAULT 0      COMMENT '父节点 ID，0 表示顶级',
  menu_name     VARCHAR(64)  NOT NULL                 COMMENT '菜单/按钮名称',
  menu_type     CHAR(1)      NOT NULL                 COMMENT '类型：M=目录, C=菜单, F=按钮',
  app_id        VARCHAR(64)  DEFAULT NULL             COMMENT '所属应用 ID，NULL 表示门户自身',
  path          VARCHAR(256) DEFAULT NULL             COMMENT '路由路径',
  component     VARCHAR(256) DEFAULT NULL             COMMENT '前端组件路径',
  perms         VARCHAR(128) DEFAULT NULL             COMMENT '权限标识',
  icon          VARCHAR(64)  DEFAULT NULL             COMMENT '菜单图标',
  sort_order    INT          NOT NULL DEFAULT 0       COMMENT '排序号',
  visible       TINYINT(1)   NOT NULL DEFAULT 1       COMMENT '是否可见：1=是, 0=否',
  is_cache      TINYINT(1)   NOT NULL DEFAULT 1       COMMENT '是否缓存：1=是, 0=否',
  is_frame      TINYINT(1)   NOT NULL DEFAULT 0       COMMENT '是否外链：1=是, 0=否',
  status        TINYINT(1)   NOT NULL DEFAULT 1       COMMENT '状态：1=正常, 0=停用',
  query_params  VARCHAR(256) DEFAULT NULL             COMMENT '路由默认查询参数 JSON',
  remark        VARCHAR(256) DEFAULT NULL             COMMENT '备注',
  created_by    BIGINT       DEFAULT NULL,
  created_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_by    BIGINT       DEFAULT NULL,
  updated_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  INDEX idx_parent_id (parent_id),
  INDEX idx_app_id (app_id),
  UNIQUE INDEX uk_perms (perms)
) COMMENT '菜单与权限资源表';
```

### 1.2 类型与层级约束

| 父节点类型        | 可挂子节点类型    | 说明                   |
| ----------------- | ----------------- | ---------------------- |
| 根（parent_id=0） | M, C              | 顶级只能是目录或菜单   |
| M（目录）         | M, C              | 目录下可嵌套目录或菜单 |
| C（菜单）         | F                 | 菜单下只能挂按钮       |
| F（按钮）         | —（不可有子节点） | 按钮为叶子节点         |

最大层级：4 级（M > M > C > F）。

### 1.3 字段校验规则

| 字段      | M（目录） | C（菜单）            | F（按钮） |
| --------- | --------- | -------------------- | --------- |
| menu_name | 必填      | 必填                 | 必填      |
| path      | 必填      | 必填                 | —         |
| component | —         | 必填（或子应用标识） | —         |
| perms     | —         | 可选                 | 必填      |
| icon      | 可选      | 可选                 | —         |
| visible   | 有效      | 有效                 | —         |
| is_cache  | —         | 有效                 | —         |
| is_frame  | —         | 有效                 | —         |

## 2. 接口规范

### 2.1 获取完整菜单树（管理员）

```
GET /menus?appId={appId}&status={status}
Authorization: Bearer {accessToken}
```

| 参数   | 必填 | 说明                         |
| ------ | ---- | ---------------------------- |
| appId  | 否   | 按应用筛选；不传返回全部     |
| status | 否   | 按状态筛选；不传返回全部状态 |

**Response 200**

```json
{
  "code": 200,
  "data": [
    {
      "id": 1,
      "parentId": 0,
      "menuName": "系统管理",
      "menuType": "M",
      "appId": null,
      "path": "/system",
      "component": null,
      "perms": null,
      "icon": "setting",
      "sortOrder": 1,
      "visible": true,
      "isCache": true,
      "isFrame": false,
      "status": 1,
      "queryParams": null,
      "children": [...]
    }
  ]
}
```

### 2.2 创建菜单

```
POST /menus
Authorization: Bearer {accessToken}
Content-Type: application/json
```

**Request Body**

```json
{
  "parentId": 1,
  "menuName": "用户管理",
  "menuType": "C",
  "appId": null,
  "path": "user",
  "component": "system/user/index",
  "perms": "user:list",
  "icon": "user",
  "sortOrder": 1,
  "visible": true,
  "isCache": true,
  "isFrame": false,
  "queryParams": null
}
```

**处理流程**

```
创建菜单
  │
  ├─ 1. 参数校验
  │      ├─ menuName 非空，长度 ≤ 64
  │      ├─ menuType 为 M / C / F 之一
  │      ├─ 根据 menuType 校验必填字段（参见 1.3 字段校验规则）
  │      └─ appId 非空时校验应用存在
  ├─ 2. parentId 校验
  │      ├─ parentId = 0 或指向已有节点
  │      ├─ 父节点类型与子节点类型兼容（参见 1.2 类型约束）
  │      └─ 层级不超过 4 级
  ├─ 3. 唯一性校验
  │      ├─ 同级下 menuName 不重复
  │      ├─ perms 全局唯一（非空时）
  │      └─ 同应用下 path 不重复（非空时）
  ├─ 4. INSERT sys_menu
  ├─ 5. 清除菜单树缓存（menu:tree、menu:tree:{appId}）
  ├─ 6. 记录操作留痕
  └─ 7. 返回新增菜单 ID
```

### 2.3 更新菜单

```
PUT /menus/{id}
Authorization: Bearer {accessToken}
Content-Type: application/json
```

**处理流程**

```
更新菜单
  │
  ├─ 1. 校验菜单存在
  ├─ 2. menuType 不可修改
  ├─ 3. 不可将自己设为自己的子节点（parentId ≠ id）
  ├─ 4. 唯一性校验（menuName、perms、path）
  ├─ 5. UPDATE sys_menu
  ├─ 6. 如果修改了 path / component / visible / status / perms
  │      ├─ 清除菜单树缓存
  │      └─ 如果修改了 perms / status → 清除相关用户权限缓存
  ├─ 7. 记录操作留痕
  └─ 8. 返回成功
```

### 2.4 删除菜单

```
DELETE /menus/{id}
Authorization: Bearer {accessToken}
```

**处理流程**

```
删除菜单
  │
  ├─ 1. 校验菜单存在
  ├─ 2. 检查是否存在子节点 → 有子节点则拒绝
  ├─ 3. 删除 sys_role_menu 中关联此菜单的记录
  ├─ 4. DELETE sys_menu
  ├─ 5. 清除受影响用户的权限缓存
  ├─ 6. 清除菜单树缓存
  ├─ 7. 记录操作留痕
  └─ 8. 返回成功
```

### 2.5 调整菜单排序（拖拽）

```
PUT /menus/{id}/sort
Authorization: Bearer {accessToken}
Content-Type: application/json
```

**Request Body**

```json
{
  "targetParentId": 1,
  "targetSortOrder": 3
}
```

**处理流程**

```
拖拽排序
  │
  ├─ 1. 校验菜单存在
  ├─ 2. 校验 targetParentId 有效且类型兼容
  ├─ 3. 校验层级不超过 4 级（含子树移动）
  ├─ 4. 事务内操作
  │      ├─ 更新 parent_id 和 sort_order
  │      └─ 重新排列同级兄弟节点的 sort_order
  ├─ 5. 清除菜单树缓存
  ├─ 6. 记录操作留痕
  └─ 7. 返回成功
```

## 3. 菜单树构建

### 3.1 后端构建逻辑

```java
// 伪代码
public List<MenuVO> getMenuTree(String appId, Integer status) {
    List<SysMenu> menus = menuMapper.selectList(appId, status);
    return buildTree(menus, 0L);
}

private List<MenuVO> buildTree(List<SysMenu> menus, Long parentId) {
    return menus.stream()
        .filter(m -> m.getParentId().equals(parentId))
        .sorted(Comparator.comparingInt(SysMenu::getSortOrder))
        .map(m -> {
            MenuVO vo = convert(m);
            vo.setChildren(buildTree(menus, m.getId()));
            return vo;
        })
        .collect(Collectors.toList());
}
```

### 3.2 缓存策略

| Key                 | 内容                                 | TTL     |
| ------------------- | ------------------------------------ | ------- |
| `menu:tree`         | 完整菜单树（管理员视角，含所有状态） | 60 分钟 |
| `menu:tree:{appId}` | 指定应用的菜单树                     | 60 分钟 |

**刷新时机**：菜单增删改、排序变更时清除。

## 4. 前端菜单管理页

### 4.1 菜单树表格

- 使用 TreeTable 组件展示菜单树
- 列：菜单名称（含图标）、类型、权限标识、路由路径、组件路径、排序号、状态、操作
- 支持展开/折叠所有
- 支持按应用筛选（下拉框：全部 / 门户 / 各子应用）
- 操作列：编辑、新增子菜单、删除、停用/启用

### 4.2 菜单编辑表单

```
表单字段联动规则：
  │
  ├─ 选择 menuType = M（目录）
  │      ├─ 显示：menuName, path, icon, sortOrder, visible
  │      └─ 隐藏：component, perms, isCache, isFrame, queryParams
  ├─ 选择 menuType = C（菜单）
  │      ├─ 显示：全部字段
  │      └─ component 联动：仅配置门户页面组件路径；子应用入口和接入配置由微前端接入模块维护
  └─ 选择 menuType = F（按钮）
         ├─ 显示：menuName, perms, sortOrder
         └─ 隐藏：path, component, icon, visible, isCache, isFrame, queryParams
```

### 4.3 拖拽排序

- 使用 sortablejs 或类似库
- 拖拽时实时预览目标位置
- 放下后调用 `PUT /menus/{id}/sort` 提交变更
- 跨层级拖拽需校验类型兼容性（前端预校验 + 后端校验）

## 5. 错误码

| 错误码 | 说明                       |
| ------ | -------------------------- |
| A0600  | 菜单不存在                 |
| A0601  | 父菜单不存在               |
| A0602  | 存在子菜单，不可删除       |
| A0603  | 同级菜单名称重复           |
| A0604  | 权限标识已存在             |
| A0605  | 同应用下路由路径重复       |
| A0606  | 菜单层级超过限制           |
| A0607  | 父节点类型不兼容           |
| A0608  | 不可将菜单设为自身的子节点 |

## 6. 安全要点

| 项目       | 要求                                               |
| ---------- | -------------------------------------------------- |
| 权限控制   | 菜单增删改需 `admin:menu:manage` 权限              |
| 类型不可变 | menuType 创建后不可修改                            |
| 层级校验   | 后端严格校验层级深度和类型兼容性                   |
| 缓存一致性 | 菜单变更后必须清除菜单树缓存和受影响用户的权限缓存 |
| 审计追踪   | 记录所有菜单的创建、修改、删除、排序操作           |
