# 06-02 动态菜单（两级加载）

## 1. 功能说明

本模块负责根据用户权限动态加载门户可访问的菜单，并展示用户可进入的子应用入口。子应用接入规则和内部路由由微前端接入机制维护；本模块侧重门户菜单权限过滤、应用入口过滤以及跨应用导航切换。

## 2. 功能清单

### 2.1 第一级：门户菜单加载（登录后立即执行）

用户登录成功后，门户加载以下内容：

- **门户自身菜单**：系统管理、个人中心等门户内置功能菜单（`app_id IS NULL` 的菜单）
- **子应用入口数据**：用户有权限访问的子应用列表（名称、图标、描述），用于应用导航栏、搜索跳转等能力
- **不加载任何子应用的具体菜单和路由**

```
登录成功
  │
  ├─ GET /menus/user → 仅返回门户自身的授权菜单
  ├─ GET /apps/user  → 返回用户有权限的子应用入口列表
  ├─ 动态注册门户自身的路由
  ├─ 渲染侧边栏（仅门户菜单）
  └─ 渲染门户首页 / 应用导航
```

### 2.2 第二级：应用上下文切换（进入子应用时执行）

用户点击子应用入口（或 URL 直接命中子应用路由前缀）时，门户侧仅完成应用上下文切换，并将路由激活交由微前端接入层处理：

- 检测当前路由是否匹配某个子应用的路由前缀（如 `/project/*`）
- 匹配到子应用 → 切换当前应用高亮、同步导航状态、激活对应子应用容器
- 未匹配到子应用 → 回到门户上下文，展示门户首页和门户菜单

```
用户进入子应用（点击入口 / URL 直接访问）
  │
  ├─ 1. 路由守卫检测路由前缀，识别目标子应用 appId
  ├─ 2. 将当前应用上下文切换为目标 appId
  ├─ 3. 同步顶部应用导航、标签页、面包屑等展示状态
  ├─ 4. 激活微前端子应用，传递 Token 和权限
  └─ 5. 子应用内部菜单和路由由子应用自行管理
```

### 2.3 子应用切换

```
从子应用 A 切换到子应用 B
  │
  ├─ 1. 子应用 A 保持挂载（或卸载，取决于微前端策略）
  ├─ 2. 切换当前应用上下文为 B
  ├─ 3. 更新导航高亮、标签页归属和应用级面包屑
  └─ 4. 激活子应用 B

从子应用返回门户
  │
  ├─ 1. 路由跳转到门户路径（如 / 或 /dashboard）
  ├─ 2. 切换回门户上下文与门户菜单
  └─ 3. 子应用可保持缓存或卸载
```

### 2.4 菜单权限过滤

- 菜单树中的每个节点关联权限标识
- 用户请求菜单时，后端自动过滤掉无权限的菜单项
- 目录下所有菜单均无权限时，目录本身也不展示
- 子应用入口列表仅展示用户至少拥有一个菜单权限的应用

## 3. 业务规则

| 编号 | 规则 | 说明 |
| --- | --- | --- |
| BR-01 | 按需激活 | 子应用入口按权限展示，进入子应用时仅切换应用上下文并激活子应用容器 |
| BR-02 | 权限过滤 | 无权限菜单不出现在用户界面和菜单 API 返回数据中 |
| BR-03 | 空目录隐藏 | 目录下无任何可见菜单时，目录本身不展示 |
| BR-04 | 应用入口过滤 | 子应用入口仅展示用户至少拥有一个菜单权限的应用 |
| BR-05 | 导航状态缓存 | 当前应用上下文、门户菜单和应用入口数据可在会话内缓存，减少重复请求 |
| BR-06 | 超管特殊处理 | 超级管理员返回所有菜单和所有子应用入口 |

## 4. 接口要求

| 接口          | 方法 | 说明                               |
| ------------- | ---- | ---------------------------------- |
| /menus/user   | GET  | 获取当前用户可访问的门户菜单       |
| /menus/routes | GET  | 获取当前用户的门户路由表           |
| /apps/user    | GET  | 获取当前用户有权限的子应用入口数据 |

## 5. 页面清单

| 页面 | 说明 |
| --- | --- |
| 门户首页 | 登录后的统一首页入口，仅展示欢迎文案；子应用入口通过应用导航等能力触达 |
