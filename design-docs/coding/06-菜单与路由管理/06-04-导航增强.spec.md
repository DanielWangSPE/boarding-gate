# 06-04 导航增强 — 编码规范（Spec）

## 1. 应用导航栏

### 1.1 布局位置

```
┌──────────────────────────────────────────────────┐
│  Logo  │  [门户]  [项目管理]  [人事管理]  ...  │ 用户头像 │
├─────────┼────────────────────────────────────────┤
│  侧边栏  │             内容区域                    │
│ (当前应用 │                                        │
│  的菜单)  │                                        │
└─────────┴────────────────────────────────────────┘
```

顶部导航栏展示可用应用列表，当前应用高亮。门户上下文展示门户菜单；进入子应用后，门户层仅保留应用级导航状态，子应用内部导航由子应用自身渲染。

### 1.2 组件实现

```vue
<!-- AppNavBar.vue -->
<template>
  <div class="app-nav-bar">
    <div class="app-logo" @click="goHome">
      <img :src="logoUrl" alt="Portal" />
    </div>
    <div
      v-for="item in navItems"
      :key="item.id"
      :class="['app-nav-item', { active: item.id === currentAppId }]"
      @click="switchTo(item)"
    >
      <icon :name="item.icon" />
      <span>{{ item.name }}</span>
    </div>
    <div class="app-nav-user">
      <!-- 用户头像、下拉菜单 -->
    </div>
  </div>
</template>

<script setup lang="ts">
const menuStore = useMenuStore();

const navItems = computed(() => {
  const portal = {
    id: null,
    name: '首页',
    icon: 'home',
    routePrefix: '/dashboard',
  };
  const apps = menuStore.userApps.map((a) => ({
    id: a.appId,
    name: a.appName,
    icon: a.icon,
    routePrefix: a.routePrefix,
  }));
  return [portal, ...apps];
});

const currentAppId = computed(() => menuStore.currentAppId);

function switchTo(item: NavItem) {
  if (item.id === currentAppId.value) return;
  router.push(item.routePrefix);
}

function goHome() {
  router.push('/dashboard');
}
</script>
```

## 2. 多标签页

### 2.1 数据结构

```typescript
interface TabItem {
  path: string; // 路由完整路径（唯一标识）
  title: string; // 显示标题
  icon?: string; // 图标
  appId: string | null; // 所属应用
  closable: boolean; // 是否可关闭
  affix: boolean; // 是否固定
}
```

### 2.2 状态管理

```typescript
const useTabStore = defineStore('tab', {
  state: () => ({
    tabs: [] as TabItem[],
    activeTab: '' as string, // 当前激活的 tab path
    cachedViews: new Set<string>(), // keep-alive 缓存的组件名
  }),

  actions: {
    // 新增标签页
    addTab(route: RouteLocationNormalized) {
      const exists = this.tabs.find((t) => t.path === route.path);
      if (exists) {
        this.activeTab = route.path;
        return;
      }

      this.tabs.push({
        path: route.path,
        title: route.meta.title || '未命名',
        icon: route.meta.icon,
        appId: route.meta.appId || null,
        closable: !route.meta.affix,
        affix: !!route.meta.affix,
      });
      this.activeTab = route.path;

      if (route.meta.keepAlive) {
        this.cachedViews.add(route.name as string);
      }
    },

    // 关闭标签页
    closeTab(path: string) {
      const idx = this.tabs.findIndex((t) => t.path === path);
      if (idx === -1 || !this.tabs[idx].closable) return;

      this.tabs.splice(idx, 1);

      // 如果关闭的是当前激活标签，激活相邻标签
      if (this.activeTab === path) {
        const next = this.tabs[idx] || this.tabs[idx - 1];
        this.activeTab = next?.path || '/dashboard';
        router.push(this.activeTab);
      }
    },

    // 关闭其他
    closeOthers(path: string) {
      this.tabs = this.tabs.filter((t) => t.affix || t.path === path);
      this.activeTab = path;
    },

    // 关闭所有
    closeAll() {
      this.tabs = this.tabs.filter((t) => t.affix);
      this.activeTab = this.tabs[0]?.path || '/dashboard';
      router.push(this.activeTab);
    },

    // 关闭左侧 / 右侧
    closeLeft(path: string) {
      const idx = this.tabs.findIndex((t) => t.path === path);
      this.tabs = this.tabs.filter((t, i) => t.affix || i >= idx);
    },
    closeRight(path: string) {
      const idx = this.tabs.findIndex((t) => t.path === path);
      this.tabs = this.tabs.filter((t, i) => t.affix || i <= idx);
    },
  },

  persist: {
    key: 'portal-tabs',
    storage: sessionStorage,
    pick: ['tabs', 'activeTab'],
  },
});
```

## 3. 面包屑导航

### 3.1 面包屑生成逻辑

```typescript
function generateBreadcrumbs(route: RouteLocationNormalized): BreadcrumbItem[] {
  const menuStore = useMenuStore();
  const breadcrumbs: BreadcrumbItem[] = [];

  // 1. 如果在子应用中，先添加应用名
  if (route.meta.appId) {
    const app = menuStore.userApps.find((a) => a.appId === route.meta.appId);
    if (app) {
      breadcrumbs.push({
        title: app.appName,
        path: app.routePrefix,
        clickable: true,
      });
    }
  }

  // 2. 门户页面根据门户菜单生成层级；子应用页面仅消费应用级前缀和子应用透出的标题
  if (!route.meta.appId) {
    const chain = findMenuChain(menuStore.portalMenus, route.path);
    chain.forEach((menu, index) => {
      breadcrumbs.push({
        title: menu.menuName,
        path: menu.path,
        clickable: index < chain.length - 1,
      });
    });
  } else if (route.meta.title) {
    breadcrumbs.push({
      title: String(route.meta.title),
      path: route.fullPath,
      clickable: false,
    });
  }

  return breadcrumbs;
}

// 递归查找菜单链
function findMenuChain(menus: MenuVO[], targetPath: string): MenuVO[] {
  for (const menu of menus) {
    if (menu.path === targetPath) return [menu];
    if (menu.children?.length) {
      const chain = findMenuChain(menu.children, targetPath);
      if (chain.length) return [menu, ...chain];
    }
  }
  return [];
}
```

## 4. 菜单搜索

### 4.1 搜索数据源

```typescript
interface SearchableMenu {
  title: string; // 菜单名称
  path: string; // 路由路径
  icon?: string; // 图标
  appId: string | null; // 所属应用
  appName: string; // 应用名称
  breadcrumb: string; // 面包屑路径，如 "项目管理 > 项目列表"
  pinyin: string; // 拼音（用于拼音搜索）
  pinyinFirst: string; // 拼音首字母
}

function buildSearchIndex(): SearchableMenu[] {
  const menuStore = useMenuStore();
  const result: SearchableMenu[] = [];

  // 门户菜单
  flattenMenus(menuStore.portalMenus, null, '门户', [], result);

  // 子应用入口
  menuStore.userApps.forEach((app) => {
    result.push({
      title: app.appName,
      path: app.routePrefix,
      icon: app.icon,
      appId: app.appId,
      appName: app.appName,
      breadcrumb: `应用入口 > ${app.appName}`,
      pinyin: toPinyin(app.appName),
      pinyinFirst: toPinyinFirst(app.appName),
    });
  });

  return result;
}
```

### 4.2 搜索匹配

```typescript
function searchMenus(
  keyword: string,
  index: SearchableMenu[],
): SearchableMenu[] {
  if (!keyword.trim()) return [];

  const kw = keyword.toLowerCase();
  return index
    .filter(
      (item) =>
        item.title.toLowerCase().includes(kw) ||
        item.pinyin.toLowerCase().includes(kw) ||
        item.pinyinFirst.toLowerCase().includes(kw),
    )
    .slice(0, 10);
}
```

## 5. 编码规范

### 5.1 组件拆分

| 组件           | 职责           | 位置                                     |
| -------------- | -------------- | ---------------------------------------- |
| AppNavBar      | 顶部应用导航   | `@/layout/components/AppNavBar.vue`      |
| SidebarMenu    | 侧边栏菜单     | `@/layout/components/SidebarMenu.vue`    |
| TabBar         | 标签页栏       | `@/layout/components/TabBar.vue`         |
| TabContextMenu | 标签页右键菜单 | `@/layout/components/TabContextMenu.vue` |
| Breadcrumb     | 面包屑         | `@/layout/components/Breadcrumb.vue`     |
| MenuSearch     | 菜单搜索弹窗   | `@/layout/components/MenuSearch.vue`     |

### 5.2 性能优化

| 优化项       | 方式                                                |
| ------------ | --------------------------------------------------- |
| 搜索索引     | 菜单变更时重建索引，搜索时直接过滤内存数据          |
| 标签页持久化 | 使用 sessionStorage，避免 localStorage 跨标签页冲突 |
| keep-alive   | 通过 `include` 精确控制缓存范围                     |
| 面包屑计算   | 使用 computed 缓存，路由变更时重新计算              |
| 拼音转换     | 菜单加载时预计算拼音，搜索时不重复转换              |

### 5.3 无障碍（Accessibility）

| 项目           | 要求                                        |
| -------------- | ------------------------------------------- |
| 标签页键盘导航 | 支持 Tab / Arrow 键在标签页间切换           |
| 搜索弹窗       | 支持 Up/Down 选择结果，Enter 跳转，Esc 关闭 |
| ARIA 属性      | 导航栏、标签页、面包屑添加合适的 ARIA role  |

## 6. 安全要点

| 项目            | 要求                                             |
| --------------- | ------------------------------------------------ |
| 标签页数据      | sessionStorage 中不存储敏感数据（仅 path/title） |
| 搜索范围        | 搜索结果仅包含用户有权限的菜单                   |
| keep-alive 清理 | 退出登录时清空所有缓存视图                       |
| 跨应用标签      | 点击跨应用标签时需校验用户仍有该应用权限         |
