# 06-03 路由联动

## 1. 功能说明

本模块负责门户平台的固定路由联动，包括门户路由守卫、导航状态同步以及与已接入子应用的激活联动。子应用路由前缀和接入规则由微前端接入机制维护，本模块仅负责消费这些配置并完成门户侧跳转与高亮。

## 2. 功能清单

### 2.1 路由分发

- 主应用维护全局路由表，包含门户自身路由和子应用路由前缀占位
- 子应用路由以路由前缀（如 `/project/*`）分组
- 路由前缀在子应用注册时配置，全局唯一，来源于微前端接入机制
- 当路由命中子应用前缀时，主应用同步应用上下文并激活子应用
- 子应用内部路由由子应用自行管理

### 2.2 路由守卫

```
全局前置守卫逻辑
  │
  ├─ 1. 检查登录态（Token 存在且有效）
  │      └─ 未登录 → 跳转登录页（记录 redirect URL）
  ├─ 2. 检查门户菜单是否已加载
  │      └─ 未加载 → 加载门户菜单 + 子应用入口列表
  ├─ 3. 判断目标路由是否属于某个子应用
  │      ├─ 是 → 同步当前应用上下文与导航状态 → 激活子应用
  │      └─ 否 → 门户自身路由，继续
  ├─ 4. 检查用户是否有权访问该路由
  │      ├─ 有权 → 放行
  │      └─ 无权 → 展示 403 页面
  └─ 5. 路由切换时显示加载动画
```

## 3. 业务规则

| 编号  | 规则         | 说明                                                  |
| ----- | ------------ | ----------------------------------------------------- |
| BR-01 | 路由唯一     | 门户自身动态路由路径不可重复                          |
| BR-02 | 前缀唯一     | 子应用路由前缀全局唯一，接入配置由微前端接入机制维护  |
| BR-03 | 未登录重定向 | 访问受保护路由时跳转登录页，记录原始 URL 供登录后回跳 |
| BR-04 | 无权限 403   | 已登录但无权访问的路由展示 403 页面                   |
| BR-05 | 子应用激活   | 路由命中子应用前缀时自动同步应用上下文并激活子应用    |
| BR-06 | 白名单路由   | 登录页、404、403 等公共页面无需权限校验               |

## 4. 接口要求

| 接口          | 方法 | 说明                     |
| ------------- | ---- | ------------------------ |
| /menus/routes | GET  | 获取当前用户的门户路由表 |

## 5. 页面清单

| 页面       | 说明                        |
| ---------- | --------------------------- |
| 403 页面   | 无权限访问提示              |
| 404 页面   | 页面不存在提示              |
| 加载过渡页 | 子应用加载时的 loading 过渡 |
