# 08-01 数据字典

## 1. 功能说明

本模块负责管理系统中所有枚举类数据的字典类型和字典项，为各业务模块提供统一的下拉选项数据源和值翻译能力。字典数据通过缓存提升访问性能，变更后主动刷新。

## 2. 功能清单

### 2.1 字典类型管理

- 字典类型名称
- 字典类型编码（唯一标识，如 `sys_user_sex`、`sys_normal_disable`）
- 字典类型状态（正常 / 停用）
- 字典类型描述

### 2.2 字典数据管理

- 字典标签（展示文本）
- 字典值（实际存储值）
- 排序号
- 样式属性（如标签颜色：primary / success / warning / danger）
- 是否默认值
- 状态（正常 / 停用）
- 备注

### 2.3 字典使用

- 前端通过字典编码获取字典项列表
- 字典数据缓存（减少重复请求）
- 字典变更后通知前端刷新缓存
- 提供字典值翻译工具方法（将存储值转为展示文本）

## 3. 业务规则

| 编号  | 规则     | 说明                                       |
| ----- | -------- | ------------------------------------------ |
| BR-01 | 编码唯一 | 字典类型编码全局唯一，创建后不可修改       |
| BR-02 | 值唯一   | 同一字典类型下字典值不可重复               |
| BR-03 | 内置保护 | 系统内置字典类型不可删除，仅允许修改字典项 |
| BR-04 | 缓存刷新 | 字典数据变更后需主动刷新 Redis 缓存        |
| BR-05 | 停用处理 | 停用的字典类型，其下所有字典项不对前端返回 |

## 4. 接口要求

| 接口              | 方法   | 说明                             |
| ----------------- | ------ | -------------------------------- |
| /dict/types       | GET    | 查询字典类型列表（分页）         |
| /dict/types       | POST   | 创建字典类型                     |
| /dict/types/{id}  | PUT    | 更新字典类型                     |
| /dict/types/{id}  | DELETE | 删除字典类型                     |
| /dict/data        | GET    | 查询字典数据列表（按类型筛选）   |
| /dict/data        | POST   | 创建字典数据                     |
| /dict/data/{id}   | PUT    | 更新字典数据                     |
| /dict/data/{id}   | DELETE | 删除字典数据                     |
| /dict/data/{code} | GET    | 按字典编码获取字典项（前端使用） |

## 5. 页面清单

| 页面       | 说明                                              |
| ---------- | ------------------------------------------------- |
| 字典管理页 | 左侧字典类型列表 + 右侧字典数据管理，支持增删改查 |
