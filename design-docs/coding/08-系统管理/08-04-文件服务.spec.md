# 08-04 文件服务 — 编码规范（Spec）

## 1. 数据模型

### 1.1 文件记录表（sys_file）

```sql
CREATE TABLE sys_file (
  id             BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
  file_name      VARCHAR(256) NOT NULL                 COMMENT '原始文件名',
  file_path      VARCHAR(512) NOT NULL                 COMMENT '存储路径',
  file_size      BIGINT       NOT NULL                 COMMENT '文件大小（字节）',
  file_type      VARCHAR(64)  NOT NULL                 COMMENT 'MIME 类型',
  file_ext       VARCHAR(16)  NOT NULL                 COMMENT '文件扩展名',
  storage_type   VARCHAR(32)  NOT NULL DEFAULT 'local' COMMENT '存储类型：local / minio',
  biz_type       VARCHAR(64)  DEFAULT NULL             COMMENT '业务类型：avatar / import / export / icon',
  biz_id         VARCHAR(64)  DEFAULT NULL             COMMENT '关联业务 ID',
  created_by     BIGINT       DEFAULT NULL,
  created_at     DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  INDEX idx_biz (biz_type, biz_id)
) COMMENT '文件记录表';
```

## 2. 接口规范

### 2.1 上传文件

```
POST /files/upload
Authorization: Bearer {accessToken}
Content-Type: multipart/form-data
```

| 参数    | 必填 | 说明                             |
| ------- | ---- | -------------------------------- |
| file    | 是   | 上传的文件                       |
| bizType | 否   | 业务类型：avatar / import / icon |
| bizId   | 否   | 关联业务 ID                      |

**Response 200**

```json
{
  "code": 200,
  "data": {
    "fileId": 1,
    "fileName": "avatar.png",
    "fileUrl": "/files/1",
    "fileSize": 102400,
    "fileType": "image/png"
  }
}
```

**处理流程**

```
文件上传
  │
  ├─ 1. 校验文件不为空
  ├─ 2. 校验文件大小
  │      └─ 读取系统参数 file.upload.maxSize（默认 10MB）
  ├─ 3. 校验文件类型（扩展名 + MIME 类型）
  │      ├─ 允许的类型白名单（可配置）
  │      │      ├─ 图片：jpg, jpeg, png, gif, webp
  │      │      ├─ 文档：pdf, doc, docx, xls, xlsx
  │      │      └─ 压缩：zip, rar
  │      └─ 拒绝可执行文件：exe, bat, sh, cmd, js, jsp, php 等
  ├─ 4. 生成存储路径
  │      └─ /{storageRoot}/{yyyy}/{MM}/{dd}/{uuid}.{ext}
  ├─ 5. 写入存储
  │      ├─ local：写入本地磁盘
  │      └─ minio：上传到 MinIO Bucket
  ├─ 6. INSERT sys_file
  └─ 7. 返回文件信息
```

### 2.2 下载文件

```
GET /files/{id}
Authorization: Bearer {accessToken}
```

**处理流程**

```
文件下载
  │
  ├─ 1. 查询 sys_file 记录
  ├─ 2. 从存储读取文件流
  ├─ 3. 设置响应头
  │      ├─ Content-Type: {fileType}
  │      ├─ Content-Disposition: attachment; filename="{原始文件名}"
  │      └─ Content-Length: {fileSize}
  └─ 4. 流式输出
```

### 2.3 删除文件

```
DELETE /files/{id}
Authorization: Bearer {accessToken}
```

**处理流程**

```
文件删除
  │
  ├─ 1. 查询 sys_file 记录
  ├─ 2. 从存储中删除物理文件
  ├─ 3. DELETE sys_file
  └─ 4. 返回成功
```

## 3. 存储后端

### 3.1 接口抽象

```java
public interface FileStorage {
    String store(InputStream inputStream, String filePath) throws IOException;
    InputStream retrieve(String filePath) throws IOException;
    void delete(String filePath) throws IOException;
    boolean exists(String filePath);
}
```

### 3.2 本地存储实现

```java
@Component
@ConditionalOnProperty(name = "file.storage.type", havingValue = "local", matchIfMissing = true)
public class LocalFileStorage implements FileStorage {

    @Value("${file.storage.local.root:/data/files}")
    private String rootPath;

    @Override
    public String store(InputStream inputStream, String filePath) throws IOException {
        Path target = Paths.get(rootPath, filePath);
        Files.createDirectories(target.getParent());
        Files.copy(inputStream, target, StandardCopyOption.REPLACE_EXISTING);
        return filePath;
    }

    @Override
    public InputStream retrieve(String filePath) throws IOException {
        return Files.newInputStream(Paths.get(rootPath, filePath));
    }

    @Override
    public void delete(String filePath) throws IOException {
        Files.deleteIfExists(Paths.get(rootPath, filePath));
    }

    @Override
    public boolean exists(String filePath) {
        return Files.exists(Paths.get(rootPath, filePath));
    }
}
```

### 3.3 MinIO 存储实现

```java
@Component
@ConditionalOnProperty(name = "file.storage.type", havingValue = "minio")
public class MinioFileStorage implements FileStorage {

    @Autowired
    private MinioClient minioClient;

    @Value("${file.storage.minio.bucket:portal-files}")
    private String bucket;

    @Override
    public String store(InputStream inputStream, String filePath) throws IOException {
        try {
            minioClient.putObject(
                PutObjectArgs.builder()
                    .bucket(bucket)
                    .object(filePath)
                    .stream(inputStream, -1, 10485760)
                    .build()
            );
            return filePath;
        } catch (Exception e) {
            throw new IOException("MinIO upload failed", e);
        }
    }

    // retrieve / delete / exists 类似实现...
}
```

## 4. 文件类型白名单

```yaml
file:
  upload:
    allowed-extensions:
      - jpg
      - jpeg
      - png
      - gif
      - webp
      - pdf
      - doc
      - docx
      - xls
      - xlsx
      - zip
      - rar
    blocked-extensions:
      - exe
      - bat
      - sh
      - cmd
      - js
      - jsp
      - php
      - py
      - class
      - jar
```

**校验逻辑**：

```java
public void validateFile(MultipartFile file) {
    // 1. 文件名扩展名校验
    String ext = getExtension(file.getOriginalFilename()).toLowerCase();
    if (blockedExtensions.contains(ext)) {
        throw new BusinessException("不允许上传该类型文件");
    }
    if (!allowedExtensions.isEmpty() && !allowedExtensions.contains(ext)) {
        throw new BusinessException("不支持的文件类型");
    }

    // 2. MIME 类型校验（防止伪造扩展名）
    String mimeType = file.getContentType();
    if (!isValidMimeType(ext, mimeType)) {
        throw new BusinessException("文件类型与扩展名不匹配");
    }

    // 3. 文件大小校验
    long maxSize = configService.getIntValue("file.upload.maxSize", 10) * 1024L * 1024L;
    if (file.getSize() > maxSize) {
        throw new BusinessException("文件大小超过限制");
    }
}
```

## 5. 文件路径生成

```java
public String generateFilePath(String originalFilename) {
    String ext = getExtension(originalFilename);
    String uuid = UUID.randomUUID().toString().replace("-", "");
    LocalDate now = LocalDate.now();
    return String.format("%d/%02d/%02d/%s.%s",
        now.getYear(), now.getMonthValue(), now.getDayOfMonth(), uuid, ext);
}

// 示例输出: 2026/04/03/a1b2c3d4e5f6789012345678abcdef01.png
```

## 7. 安全要点

| 项目     | 要求                                      |
| -------- | ----------------------------------------- |
| 类型校验 | 扩展名 + MIME 类型双重校验，防止伪造      |
| 大小限制 | 通过系统参数配置，可运行时调整            |
| 路径安全 | UUID 重命名，不保留原始文件名作为存储路径 |
| 目录穿越 | 存储路径不拼接用户输入，防止 `../` 穿越   |
| 访问控制 | 下载文件需登录态                          |
| 物理清理 | 删除记录时同步删除物理文件                |
