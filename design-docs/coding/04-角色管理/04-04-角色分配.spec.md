# 04-04 角色分配 — 编码规范（Spec）

## 1. 数据模型

复用 `sys_user_role` 表（参见 02-03 用户关联关系 spec）。

角色有效期预留字段（一期不实现回收逻辑）：

```sql
ALTER TABLE sys_user_role ADD COLUMN expire_at DATETIME NULL COMMENT '角色有效期（到期自动回收），NULL 表示永久';
```

## 2. 接口规范

### 2.1 查询角色下的用户列表

```
GET /roles/{id}/users?keyword=&deptId=&page=1&pageSize=20
Authorization: Bearer {accessToken}
```

**Response 200**

```json
{
  "code": 200,
  "data": {
    "total": 50,
    "list": [
      {
        "userId": 1,
        "username": "zhangsan",
        "nickname": "张三",
        "department": "技术部",
        "status": 1,
        "assignedAt": "2026-01-15T10:00:00Z",
        "expireAt": null
      }
    ]
  }
}
```

### 2.2 批量向角色添加用户

```
POST /roles/{id}/users
Authorization: Bearer {accessToken}
Content-Type: application/json
```

**Request Body**

```json
{
  "userIds": [10, 11, 12]
}
```

**Response 200**

```json
{
  "code": 200,
  "data": {
    "addedCount": 2,
    "skippedCount": 1
  }
}
```

**处理流程**

```
批量添加用户到角色
  │
  ├─ 1. 校验角色存在且状态正常
  ├─ 2. 校验所有 userIds 存在且未注销
  ├─ 3. 过滤已拥有该角色的用户（跳过，不报错）
  ├─ 4. 批量插入 sys_user_role
  ├─ 5. 批量清除成功用户的权限缓存
  ├─ 6. 记录操作留痕（异步）
  └─ 7. 返回结果
```

### 2.3 批量从角色移除用户

```
DELETE /roles/{id}/users
Authorization: Bearer {accessToken}
Content-Type: application/json
```

**Request Body**

```json
{
  "userIds": [10, 11]
}
```

**处理流程**

```
批量移除用户
  │
  ├─ 1. 校验角色存在
  ├─ 2. 如果是超级管理员角色，检查移除后是否仍有至少一个用户
  ├─ 3. 删除 sys_user_role 中的关联记录
  ├─ 4. 批量清除这些用户的权限缓存
  ├─ 5. 记录操作留痕（异步）
  └─ 6. 返回成功
```

### 2.4 按部门批量分配

```
POST /roles/{id}/users/by-dept
Authorization: Bearer {accessToken}
Content-Type: application/json
```

**Request Body**

```json
{
  "deptId": 10,
  "includeChildren": true
}
```

**处理流程**

```
按部门分配角色
  │
  ├─ 1. 校验角色存在且状态正常
  ├─ 2. 查询部门下所有用户 ID（含/不含下级部门）
  ├─ 3. 过滤已拥有该角色的用户
  ├─ 4. 批量插入 sys_user_role
  ├─ 5. 批量清除权限缓存
  ├─ 6. 记录操作留痕
  └─ 7. 返回结果
```

## 3. 编码规范

### 3.1 角色维度 vs 用户维度

| 维度 | 操作入口 | 差异 |
| --- | --- | --- |
| 角色维度（本模块） | 角色详情 → 用户列表 | 一次操作一个角色的多个用户，支持按部门批量分配 |
| 用户维度（02-03 模块） | 用户详情 → 角色分配 | 一次操作一个用户的多个角色，全量替换模式 |

两个维度操作同一张 `sys_user_role` 表，核心逻辑（角色校验、缓存清除）复用同一个 Service 方法。

### 3.2 超级管理员保护

- 超级管理员角色的用户关联仅允许拥有超级管理员角色的用户操作
- 移除超级管理员角色的用户时，检查至少保留一个超级管理员用户
- 不可移除自己的超级管理员角色

### 3.3 按部门分配性能

- 大部门可能包含几百上千用户
- 批量插入使用 `INSERT IGNORE`（已有关联的自动跳过）
- 缓存清除使用 Redis Pipeline

### 3.4 角色有效期（预留）

- `expire_at` 字段一期写入但不消费
- 二期实现后台任务：扫描 `expire_at < NOW()` 的记录，自动删除并清除权限缓存
- 分配角色时可选传入 `expireAt` 参数

## 4. 安全要点

| 项目       | 要求                                |
| ---------- | ----------------------------------- |
| 权限控制   | 角色分配需 `admin:role:assign` 权限 |
| 超管保护   | 超级管理员角色的分配有额外约束      |
| 缓存一致性 | 分配变更后批量清除权限缓存          |
| 操作留痕   | 记录操作人、角色、受影响用户列表    |
