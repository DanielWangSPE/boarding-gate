# 05-01 功能权限

## 1. 功能说明

本模块负责管理权限资源的定义（菜单、按钮），以及通过权限标识实现前端 UI 控制和后端接口鉴权。权限资源按应用隔离，各子应用注册自己的权限资源，门户统一聚合管理。

## 2. 功能清单

### 2.1 权限资源管理

- 定义权限资源（菜单、按钮）
- 权限资源按应用隔离（各子应用管理自己的权限资源，门户统一聚合）
- 权限资源以树形结构组织
- 支持权限资源的批量导入（子应用接入时注册权限资源）

### 2.2 菜单权限

- 控制用户可见的菜单项
- 无权限的菜单不在界面中展示
- 菜单权限与角色关联

### 2.3 按钮权限

- 控制页面内的按钮可见性
- 权限标识（如 `user:create`、`user:delete`）
- 前端通过权限指令 / 组件控制按钮渲染
- 后端 API 接口通过同一权限标识鉴权，前后端保持一致
- 未授权的 API 请求返回 403 Forbidden
- 公共接口（如登录、获取验证码）设为白名单，无需鉴权

## 3. 业务规则

| 编号  | 规则         | 说明                                             |
| ----- | ------------ | ------------------------------------------------ |
| BR-01 | 权限并集     | 用户最终权限 = 所有角色权限的并集                |
| BR-02 | 拒绝优先     | 存在明确拒绝策略时，拒绝优先于允许               |
| BR-03 | 超管豁免     | 超级管理员拥有所有权限（`*:*:*`），不受限制      |
| BR-04 | 前后一致     | 前端按钮权限标识与后端接口鉴权标识为同一个       |
| BR-05 | 子应用隔离   | 各子应用的权限资源相互隔离，但可在角色中聚合分配 |
| BR-06 | 权限标识规范 | 格式 `模块:操作`，如 `project:create`，全局唯一  |

## 4. 接口要求

| 接口                        | 方法   | 说明                           |
| --------------------------- | ------ | ------------------------------ |
| /permissions                | GET    | 获取权限资源树                 |
| /permissions                | POST   | 创建权限资源                   |
| /permissions/{id}           | PUT    | 更新权限资源                   |
| /permissions/{id}           | DELETE | 删除权限资源                   |
| /permissions/apps/{appId}   | GET    | 获取指定应用的权限资源         |
| /permissions/batch-register | POST   | 批量注册权限资源（子应用接入） |

## 5. 页面清单

| 页面           | 说明                                                 |
| -------------- | ---------------------------------------------------- |
| 权限资源管理页 | 以树形结构管理权限资源（菜单、按钮），支持按应用筛选 |
| 权限分配总览页 | 按角色或用户维度查看权限分配情况                     |
