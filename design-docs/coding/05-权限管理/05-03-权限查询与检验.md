# 05-03 权限查询与检验

## 1. 功能说明

本模块提供统一的权限查询服务和前端权限工具，供主应用和子应用调用。包括后端 API 查询接口和前端 SDK（权限指令、组件、Hook），支持权限数据缓存以提升查询性能。

## 2. 功能清单

### 2.1 权限查询服务

- 提供统一的权限查询 API，供主应用和子应用调用
- 查询当前用户的完整权限列表（功能权限标识集合）
- 查询当前用户是否拥有指定权限
- 查询当前用户的数据权限范围
- 权限数据支持 Redis 缓存，提升查询性能
- 角色权限变更时主动刷新受影响用户的缓存

### 2.2 前端权限工具（SDK）

- 提供权限指令（如 `v-permission`），控制元素显隐
- 提供权限组件（如 `<Authorized>`），包裹需要权限控制的内容
- 提供权限 Hook（如 `usePermission`），供逻辑层判断
- 以 SDK / npm 包形式提供给子应用使用
- 权限数据由主应用加载后传递给子应用，子应用不重复请求

## 3. 业务规则

| 编号 | 规则 | 说明 |
| --- | --- | --- |
| BR-01 | 权限缓存 | 权限数据使用 Redis 缓存，角色权限变更时主动刷新 |
| BR-02 | 缓存粒度 | 按用户维度缓存（`permission:user:{userId}`、`menu:user:{userId}`） |
| BR-03 | 超管快速路径 | 超级管理员直接返回 `*:*:*`，不查询权限表 |
| BR-04 | 前端本地缓存 | 前端登录后一次性拉取权限集，存于内存，退出清空 |
| BR-05 | SDK 统一 | 子应用通过主应用提供的 SDK 进行权限判断，不直接调用接口 |

## 4. 接口要求

| 接口                    | 方法 | 说明                         |
| ----------------------- | ---- | ---------------------------- |
| /permissions/check      | POST | 校验当前用户是否具有指定权限 |
| /permissions/current    | GET  | 获取当前用户的完整权限集     |
| /permissions/data-scope | GET  | 获取当前用户的数据权限范围   |
