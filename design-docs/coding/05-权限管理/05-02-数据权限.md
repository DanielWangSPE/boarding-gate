# 05-02 数据权限

## 1. 功能说明

本模块负责控制用户可访问的数据范围，包括行级数据权限（控制可见数据行）和列级数据权限（控制可见数据字段）。数据权限在数据库查询层面自动注入过滤条件，实现业务代码零侵入。

## 2. 功能清单

### 2.1 行级数据权限

- 控制用户可访问的数据行范围
- 预设数据范围：
  - **全部数据**（ALL）：无限制
  - **本部门数据**（DEPT）：仅可见所属部门的数据
  - **本部门及下级数据**（DEPT_TREE）：可见所属部门及其下级部门的数据
  - **仅本人数据**（SELF）：仅可见自己创建的数据
  - **自定义范围**（CUSTOM）：指定特定部门列表
- 数据权限与角色关联，通过角色的 `data_scope` 字段配置
- 多角色场景下，取最大数据范围（ALL > CUSTOM/DEPT_TREE > DEPT > SELF）
- 数据权限在数据库查询层面自动注入过滤条件（业务代码零侵入）

### 2.2 列级数据权限（扩展）

- 控制用户可见的数据字段
- 如：普通用户不可查看工资字段
- 前端渲染和 API 返回同时受控
- 以注解或配置方式标记受控字段

## 3. 业务规则

| 编号 | 规则 | 说明 |
| --- | --- | --- |
| BR-01 | 范围取大 | 多角色取最大数据范围：ALL > CUSTOM/DEPT_TREE > DEPT > SELF |
| BR-02 | 自动注入 | 通过 AOP / MyBatis 拦截器自动在 SQL 中注入数据过滤条件 |
| BR-03 | 超管豁免 | 超级管理员不受数据权限限制，等同于 ALL |
| BR-04 | 部门依赖 | 数据范围依赖组织架构的部门层级关系 |
| BR-05 | 创建人标记 | SELF 范围依赖数据表中的 `create_by` 字段 |
| BR-06 | 零侵入 | 业务代码中不需要编写数据权限过滤逻辑 |

## 4. 接口要求

| 接口                    | 方法 | 说明                       |
| ----------------------- | ---- | -------------------------- |
| /permissions/data-scope | GET  | 获取当前用户的数据权限范围 |

> 数据权限的配置通过角色管理模块的 `PUT /roles/{id}/data-scope` 接口完成，此处仅提供查询能力。
