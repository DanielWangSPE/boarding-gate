# 07-05 应用市场 — 编码规范（Spec）

> 本模块为扩展功能，一期可不实现，此处提供设计规范供后续参考。

## 1. 数据模型

### 1.1 用户应用订阅表（sys_user_app）

```sql
CREATE TABLE sys_user_app (
  id         BIGINT      NOT NULL AUTO_INCREMENT COMMENT '主键',
  user_id    BIGINT      NOT NULL                 COMMENT '用户 ID',
  app_id     VARCHAR(64) NOT NULL                 COMMENT '应用标识',
  sort_order INT         NOT NULL DEFAULT 0       COMMENT '个人排序',
  pinned     TINYINT(1)  NOT NULL DEFAULT 0       COMMENT '是否置顶',
  created_at DATETIME    NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  UNIQUE INDEX uk_user_app (user_id, app_id)
) COMMENT '用户应用订阅表';
```

### 1.2 应用使用统计表（sys_app_stats）

```sql
CREATE TABLE sys_app_stats (
  id         BIGINT      NOT NULL AUTO_INCREMENT COMMENT '主键',
  app_id     VARCHAR(64) NOT NULL                 COMMENT '应用标识',
  stat_date  DATE        NOT NULL                 COMMENT '统计日期',
  pv         INT         NOT NULL DEFAULT 0       COMMENT '页面访问量',
  uv         INT         NOT NULL DEFAULT 0       COMMENT '独立访客数',
  avg_duration INT       DEFAULT NULL             COMMENT '平均使用时长（秒）',
  PRIMARY KEY (id),
  UNIQUE INDEX uk_app_date (app_id, stat_date)
) COMMENT '应用使用统计日表';
```

## 2. 接口规范

### 2.1 获取应用市场列表

```
GET /app-market?keyword={keyword}&category={category}
Authorization: Bearer {accessToken}
```

**Response 200**

```json
{
  "code": 200,
  "data": [
    {
      "appId": "project-management",
      "appName": "项目管理",
      "description": "项目全生命周期管理系统",
      "icon": "project",
      "screenshots": [
        "/screenshots/project-1.png",
        "/screenshots/project-2.png"
      ],
      "category": "业务系统",
      "subscribed": true,
      "subscriberCount": 128,
      "mandatory": false
    }
  ]
}
```

### 2.2 订阅应用

```
POST /app-market/subscribe
Authorization: Bearer {accessToken}
Content-Type: application/json
```

```json
{ "appId": "project-management" }
```

**处理流程**

```
订阅应用
  │
  ├─ 1. 校验应用存在且状态正常
  ├─ 2. 校验用户对该应用有角色权限（至少一个菜单权限）
  ├─ 3. 校验未重复订阅
  ├─ 4. INSERT sys_user_app
  ├─ 5. 清除用户的 apps:user:{userId} 缓存
  └─ 6. 返回成功
```

### 2.3 取消订阅

```
POST /app-market/unsubscribe
Authorization: Bearer {accessToken}
Content-Type: application/json
```

```json
{ "appId": "project-management" }
```

**处理流程**

```
取消订阅
  │
  ├─ 1. 校验应用非强制订阅（mandatory = false）
  ├─ 2. DELETE sys_user_app WHERE user_id = ? AND app_id = ?
  ├─ 3. 清除用户的 apps:user:{userId} 缓存
  └─ 4. 返回成功
```

### 2.4 获取我的订阅

```
GET /app-market/my
Authorization: Bearer {accessToken}
```

### 2.5 获取应用使用统计

```
GET /app-market/stats/{appId}?range=30d
Authorization: Bearer {accessToken}
```

**Response 200**


## 3. 订阅与应用入口的关系

```
用户的子应用入口列表（GET /apps/user）
  │
  ├─ 一期（无应用市场）
  │      └─ 返回用户有权限的所有子应用
  │
  └─ 二期（有应用市场）
         ├─ 如果 sys_user_app 中该用户有订阅记录
         │      └─ 返回已订阅的应用（取交集：有权限 ∩ 已订阅）
         ├─ 如果 sys_user_app 中该用户无任何记录（新用户）
         │      └─ 返回所有有权限的应用（兼容未使用应用市场的用户）
         └─ 强制应用始终返回（不受订阅状态影响）
```

## 4. 统计数据采集

```
前端埋点
  │
  ├─ 子应用 mounted 时记录访问（PV +1）
  ├─ 按 userId 去重计算 UV
  ├─ 子应用 unmount 时计算使用时长
  ├─ 通过 @portal/tracking-sdk 上报
  └─ 后端按日聚合写入 sys_app_stats
```

## 5. 前端页面

### 5.1 应用市场页

```
布局：网格卡片
  ├─ 搜索栏 + 分类筛选
  ├─ 应用卡片（图标、名称、描述、订阅人数、订阅按钮）
  ├─ 点击卡片 → 应用详情（截图、使用说明、订阅/取消）
  └─ 已订阅的应用显示已订阅标记
```

## 6. 错误码

| 错误码 | 说明                             |
| ------ | -------------------------------- |
| A0750  | 应用不存在或已停用               |
| A0751  | 无权订阅该应用（无任何菜单权限） |
| A0752  | 已订阅，无需重复操作             |
| A0753  | 该应用为强制应用，不可取消订阅   |

## 7. 安全要点

| 项目     | 要求                         |
| -------- | ---------------------------- |
| 权限前提 | 用户只能订阅有权限的应用     |
| 强制应用 | 管理员标记的强制应用不可取消 |
| 统计隐私 | 使用统计仅管理员可查看       |
| 截图安全 | 应用截图通过内部文件服务存储 |
