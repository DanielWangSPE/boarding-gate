# 07-01 子应用注册与管理

## 1. 功能说明

本模块负责子应用的注册、信息维护、状态管理和健康检查。管理员通过管理界面注册新的子应用，配置入口地址和路由前缀等信息，并对子应用的运行状态进行监控。

## 2. 功能清单

### 2.1 子应用信息

- 应用名称
- 应用标识（唯一编码，注册后不可修改）
- 应用描述
- 应用图标
- 入口地址（Entry URL）
- 激活路由前缀（如 `/project`，全局唯一）
- 应用状态（正常 / 停用 / 维护中）
- 应用版本号
- 应用负责人（关联用户）
- 接入时间
- 部署环境配置（开发/测试/生产环境对应的入口地址）

### 2.2 子应用操作

- 子应用的注册、编辑、下线
- 子应用状态管理（正常 → 维护中 → 停用）
- 维护中状态：用户访问时展示维护提示页
- 停用状态：菜单中不展示，路由不可访问
- 子应用健康检查（定时探测入口地址可用性）

## 3. 业务规则

| 编号 | 规则 | 说明 |
| --- | --- | --- |
| BR-01 | 应用标识唯一 | 子应用标识全局唯一，注册后不可修改 |
| BR-02 | 路由前缀唯一 | 各子应用的激活路由前缀不可重复，不可与门户自身路由冲突 |
| BR-03 | 下线限制 | 存在关联菜单或角色的子应用不可直接下线，需先解除关联 |
| BR-04 | 健康检查 | 子应用入口地址不可用时自动标记为异常状态 |
| BR-05 | 环境隔离 | 不同部署环境使用各自配置的入口地址 |

## 4. 接口要求

| 接口              | 方法   | 说明                               |
| ----------------- | ------ | ---------------------------------- |
| /apps             | GET    | 查询子应用列表（管理员，分页）     |
| /apps             | POST   | 注册子应用                         |
| /apps/{id}        | GET    | 获取子应用详情                     |
| /apps/{id}        | PUT    | 更新子应用信息                     |
| /apps/{id}        | DELETE | 下线子应用                         |
| /apps/{id}/status | PUT    | 变更子应用状态                     |
| /apps/{id}/health | GET    | 子应用健康检查                     |
| /apps/user        | GET    | 获取当前用户有权限的子应用入口列表 |

## 5. 页面清单

| 页面              | 说明                                     |
| ----------------- | ---------------------------------------- |
| 子应用管理页      | 管理所有注册的子应用，展示状态和健康指标 |
| 子应用注册/编辑页 | 新增/编辑子应用信息表单                  |
| 子应用监控页      | 查看子应用健康状态和运行指标             |
| 维护中提示页      | 子应用处于维护中时的用户提示             |
