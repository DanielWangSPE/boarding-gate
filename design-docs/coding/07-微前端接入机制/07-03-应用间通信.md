# 07-03 应用间通信

## 1. 功能说明

本模块定义主应用与子应用、子应用与子应用之间的通信机制和协议规范。基于 micro-app 内置的数据通信和事件通信能力，提供类型安全的通信 SDK。

## 2. 功能清单

### 2.1 主应用 → 子应用

- 主应用向子应用传递全局状态（用户信息、Token、权限信息、主题配置等）
- 主应用通知子应用执行特定操作（如登出清理、主题切换）
- 通过 micro-app 的 `setData` API 传递数据

### 2.2 子应用 → 主应用

- 子应用向主应用发送事件（如路由变更、打开全局弹窗、修改页面标题）
- 子应用请求主应用提供的公共能力（如打开全局弹窗、触发统一登出）
- 通过 micro-app 的 `dispatch` API 发送事件

### 2.3 子应用 → 子应用

- 跨子应用的事件通信（通过主应用中转）
- 跨子应用的路由跳转

### 2.4 通信协议约定

- 定义标准事件类型（如 `route-change`、`logout`、`theme-change`）
- 提供类型安全的通信 SDK（TypeScript 类型定义）
- 通信事件的订阅和取消订阅管理
- 子应用卸载时 micro-app 自动清理数据监听，防止泄漏

## 3. 业务规则

| 编号 | 规则 | 说明 |
| --- | --- | --- |
| BR-01 | 类型安全 | 通信数据需定义 TypeScript 类型，避免拼写错误和类型不匹配 |
| BR-02 | 自动清理 | 子应用卸载时自动清理所有数据监听 |
| BR-03 | 主应用中转 | 子应用间通信必须通过主应用中转，不可直接通信 |
| BR-04 | 单向数据流 | 全局状态由主应用下发，子应用不可直接修改主应用状态 |
