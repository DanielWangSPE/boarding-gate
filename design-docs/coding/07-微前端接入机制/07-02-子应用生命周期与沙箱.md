# 07-02 子应用生命周期与沙箱

## 1. 功能说明

本模块负责子应用的运行时管理，包括生命周期钩子（加载、挂载、卸载）、加载策略（按需加载、预加载、资源缓存）、异常处理，以及 JS 沙箱、CSS 隔离、运行时隔离等沙箱机制。基于 micro-app 框架实现。

## 2. 功能清单

### 2.1 生命周期钩子（micro-app）

micro-app 的子应用无需导出生命周期函数，框架通过自定义事件通知子应用：

- **created**：`<micro-app>` 标签被创建
- **beforemount**：子应用资源加载完成，即将渲染
- **mounted**：子应用渲染完成
- **unmount**：子应用卸载
- **error**：子应用加载或渲染出错

### 2.2 加载策略

- 首次访问时加载子应用资源
- 预加载策略：空闲时预加载高频使用的子应用
- 资源缓存：子应用资源加载后缓存，再次激活时无需重新下载
- keep-alive：子应用切走时冻结状态，切回时恢复
- 加载超时处理：加载超时展示友好提示，支持重试
- 加载过程展示 Loading 状态

### 2.3 异常处理

- 子应用加载失败时的降级处理（展示错误页面，不影响主应用和其他子应用）
- 子应用运行时异常捕获和上报
- 子应用渲染异常时的错误边界（Error Boundary）

### 2.4 JS 沙箱

- 子应用的全局变量隔离，避免子应用污染全局环境
- 子应用卸载时自动恢复全局状态
- 支持多个子应用同时运行（如弹窗场景）

### 2.5 CSS 隔离

- 子应用的样式隔离，避免样式相互影响
- 默认使用 Shadow DOM 实现样式隔离
- 主应用的全局样式（如主题 CSS 变量）可穿透到子应用

### 2.6 运行时隔离

- 子应用之间的 DOM 隔离
- 子应用的路由隔离（子应用路由变化不影响其他子应用）
- 子应用的存储隔离（localStorage / sessionStorage 按应用加前缀）

## 3. 业务规则

| 编号  | 规则       | 说明                                              |
| ----- | ---------- | ------------------------------------------------- |
| BR-01 | 故障隔离   | 子应用异常不得影响主应用和其他子应用的正常运行    |
| BR-02 | 样式隔离   | 子应用样式变化不得影响主应用和其他子应用          |
| BR-03 | 资源缓存   | 子应用资源首次加载后缓存，后续激活复用            |
| BR-04 | keep-alive | 子应用切走时冻结、切回时恢复，保留用户操作状态    |
| BR-05 | 超时重试   | 子应用加载超时后展示提示，支持用户手动重试        |
| BR-06 | 存储隔离   | 各子应用的 localStorage / sessionStorage 互不干扰 |
