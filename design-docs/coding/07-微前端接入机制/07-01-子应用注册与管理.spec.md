# 07-01 子应用注册与管理 — 编码规范（Spec）

## 1. 数据模型

### 1.1 子应用表（sys_app）

```sql
CREATE TABLE sys_app (
  id            BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
  app_id        VARCHAR(64)  NOT NULL                 COMMENT '应用标识（唯一，注册后不可改）',
  app_name      VARCHAR(128) NOT NULL                 COMMENT '应用名称',
  description   VARCHAR(512) DEFAULT NULL             COMMENT '应用描述',
  icon          VARCHAR(256) DEFAULT NULL             COMMENT '应用图标',
  route_prefix  VARCHAR(128) NOT NULL                 COMMENT '激活路由前缀，如 /project',
  status        TINYINT(1)   NOT NULL DEFAULT 1       COMMENT '状态：1=正常, 2=维护中, 0=停用',
  version       VARCHAR(32)  DEFAULT NULL             COMMENT '当前版本号',
  owner_id      BIGINT       DEFAULT NULL             COMMENT '应用负责人 ID',
  sort_order    INT          NOT NULL DEFAULT 0       COMMENT '排序号',
  health_status TINYINT(1)   NOT NULL DEFAULT 1       COMMENT '健康状态：1=正常, 0=异常',
  health_check_at DATETIME   DEFAULT NULL             COMMENT '最近一次健康检查时间',
  remark        VARCHAR(256) DEFAULT NULL             COMMENT '备注',
  created_by    BIGINT       DEFAULT NULL,
  created_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_by    BIGINT       DEFAULT NULL,
  updated_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  UNIQUE INDEX uk_app_id (app_id),
  UNIQUE INDEX uk_route_prefix (route_prefix)
) COMMENT '子应用注册表';
```

### 1.2 子应用环境配置表（sys_app_env）

```sql
CREATE TABLE sys_app_env (
  id            BIGINT       NOT NULL AUTO_INCREMENT COMMENT '主键',
  app_id        VARCHAR(64)  NOT NULL                 COMMENT '应用标识',
  env           VARCHAR(32)  NOT NULL                 COMMENT '环境标识：dev/test/staging/prod',
  entry_url     VARCHAR(512) NOT NULL                 COMMENT '子应用入口地址',
  is_active     TINYINT(1)   NOT NULL DEFAULT 1       COMMENT '是否启用',
  PRIMARY KEY (id),
  UNIQUE INDEX uk_app_env (app_id, env)
) COMMENT '子应用环境配置表';
```

### 1.3 状态枚举

```java
public enum AppStatus {
    DISABLED(0, "停用"),
    NORMAL(1, "正常"),
    MAINTENANCE(2, "维护中");
}
```

## 2. 接口规范

### 2.1 注册子应用

```
POST /apps
Authorization: Bearer {accessToken}
Content-Type: application/json
```

**Request Body**

```json
{
  "appId": "project-management",
  "appName": "项目管理",
  "description": "项目全生命周期管理系统",
  "icon": "project",
  "routePrefix": "/project",
  "version": "1.0.0",
  "ownerId": 100,
  "sortOrder": 1,
  "envConfigs": [
    { "env": "dev", "entryUrl": "http://localhost:8081" },
    { "env": "prod", "entryUrl": "http://project.internal.com" }
  ]
}
```

**处理流程**

```
注册子应用
  │
  ├─ 1. 参数校验
  │      ├─ appId 必填，格式 ^[a-z][a-z0-9-]{1,62}[a-z0-9]$
  │      ├─ appName 必填，长度 ≤ 128
  │      ├─ routePrefix 必填，格式 ^/[a-z][a-z0-9-]*$
  │      └─ envConfigs 至少包含一个环境
  ├─ 2. 唯一性校验
  │      ├─ appId 不重复
  │      ├─ routePrefix 不重复
  │      └─ routePrefix 不与门户内置路由冲突（/login, /dashboard 门户首页, /system 等）
  ├─ 3. 校验 ownerId 对应的用户存在
  ├─ 4. 事务内操作
  │      ├─ INSERT sys_app
  │      └─ 批量 INSERT sys_app_env
  ├─ 5. 清除子应用列表缓存
  ├─ 6. 记录操作留痕
  └─ 7. 返回新增应用 ID
```

### 2.4 更新子应用

```
PUT /apps/{id}
Authorization: Bearer {accessToken}
```

**注意**：`appId` 和 `routePrefix` 注册后不可修改（防止引发菜单和路由失效）。

### 2.5 变更子应用状态

```
PUT /apps/{id}/status
Authorization: Bearer {accessToken}
Content-Type: application/json
```

```json
{ "status": 2, "reason": "系统升级维护" }
```

**处理流程**

```
变更状态
  │
  ├─ 1. 校验应用存在
  ├─ 2. 状态转换校验
  │      ├─ 正常 → 维护中 / 停用 ✅
  │      ├─ 维护中 → 正常 / 停用 ✅
  │      ├─ 停用 → 正常 ✅
  │      └─ 其他 → 拒绝
  ├─ 3. UPDATE sys_app.status
  ├─ 4. 如果变为停用
  │      ├─ 清除该应用的菜单缓存
  │      └─ 清除所有用户的 apps:user:* 缓存
  ├─ 5. 记录操作留痕（含变更原因）
  └─ 6. 返回成功
```

### 2.6 下线子应用

```
DELETE /apps/{id}
Authorization: Bearer {accessToken}
```

**处理流程**

```
下线子应用
  │
  ├─ 1. 校验应用存在
  ├─ 2. 前置检查
  │      ├─ 检查是否存在关联菜单 → 有则拒绝（需先删除菜单）
  │      └─ 检查是否存在关联角色 → 有则拒绝（需先解除角色关联）
  ├─ 3. 事务内操作
  │      ├─ DELETE sys_app_env（该应用的环境配置）
  │      └─ DELETE sys_app
  ├─ 4. 清除相关缓存
  ├─ 5. 记录操作留痕
  └─ 6. 返回成功
```

## 3. 健康检查机制

### 3.1 后台健康检查任务

```
后台健康检查任务（每 5 分钟执行一次，可配置）
  │
  ├─ 查询所有状态=正常或维护中的子应用
  ├─ 对每个应用的当前环境 entryUrl 发起 HTTP HEAD 请求
  │      ├─ 超时时间：5 秒
  │      ├─ 响应 2xx → 健康
  │      └─ 超时 / 非 2xx / 异常 → 不健康
  ├─ 更新 sys_app.health_status 和 health_check_at
  ├─ 连续 3 次不健康 → 标记为异常状态
  │      └─ 发送通知给应用负责人
  └─ 恢复健康 → 清除异常标记
```

### 3.2 配置项

| 配置 Key                         | 默认值    | 说明             |
| -------------------------------- | --------- | ---------------- |
| `app.health.check.interval`      | 300（秒） | 健康检查间隔     |
| `app.health.check.timeout`       | 5（秒）   | 单次检查超时     |
| `app.health.check.failThreshold` | 3         | 连续失败次数阈值 |

## 4. 入口地址环境解析

```java
// 根据当前部署环境获取子应用入口地址
public String resolveEntryUrl(String appId) {
    String currentEnv = appConfig.getActiveEnv(); // dev / test / prod
    SysAppEnv envConfig = appEnvMapper.selectByAppIdAndEnv(appId, currentEnv);

    if (envConfig == null || !envConfig.getIsActive()) {
        throw new BusinessException("子应用 " + appId + " 未配置 " + currentEnv + " 环境入口");
    }

    return envConfig.getEntryUrl();
}
```

前端通过 `/apps/user` 接口获取的 `entryUrl` 已经是当前环境对应的地址。

## 5. 缓存策略

| Key                  | 内容                         | TTL     |
| -------------------- | ---------------------------- | ------- |
| `app:list`           | 全部子应用列表（管理员视角） | 30 分钟 |
| `apps:user:{userId}` | 用户有权限的子应用入口列表   | 30 分钟 |
| `app:detail:{appId}` | 子应用详情                   | 30 分钟 |

## 6. 错误码

| 错误码 | 说明                       |
| ------ | -------------------------- |
| A0700  | 子应用不存在               |
| A0701  | 应用标识已存在             |
| A0702  | 路由前缀已存在             |
| A0703  | 路由前缀与门户内置路由冲突 |
| A0704  | 存在关联菜单，不可下线     |
| A0705  | 存在关联角色，不可下线     |
| A0706  | 应用标识不可修改           |
| A0707  | 路由前缀不可修改           |
| A0708  | 状态转换不合法             |
| A0709  | 未配置当前环境的入口地址   |

## 7. 安全要点

| 项目               | 要求                                           |
| ------------------ | ---------------------------------------------- |
| 权限控制           | 子应用管理需 `admin:app:manage` 权限           |
| appId 不可变       | 注册后不可修改，防止菜单和权限关联断裂         |
| routePrefix 不可变 | 注册后不可修改，防止路由分发失效               |
| 入口地址安全       | 入口地址需为内网可访问地址，校验格式合法性     |
| 审计追踪           | 记录所有子应用的注册、修改、状态变更、下线操作 |
