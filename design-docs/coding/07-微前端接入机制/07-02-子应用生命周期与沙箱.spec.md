# 07-02 子应用生命周期与沙箱 — 编码规范（Spec）

## 1. micro-app 初始化

### 1.1 主应用入口配置

```typescript
// main.ts
import microApp from '@micro-zoe/micro-app';

microApp.start({
  // 预加载配置
  prefetchApps: [
    { name: 'project', url: 'http://project.internal.com', level: 2 },
  ],
  // 全局生命周期
  lifeCycles: {
    created(e) {
      console.log(`[micro-app] ${e.detail.name} created`);
    },
    beforemount(e) {
      console.log(`[micro-app] ${e.detail.name} beforemount`);
    },
    mounted(e) {
      console.log(`[micro-app] ${e.detail.name} mounted`);
    },
    unmount(e) {
      console.log(`[micro-app] ${e.detail.name} unmount`);
    },
    error(e) {
      console.error(`[micro-app] ${e.detail.name} error`, e.detail.error);
      reportAppError(e.detail.name, e.detail.error);
    },
  },
});
```

## 2. 生命周期管理

### 2.1 子应用容器组件

```vue
<!-- SubAppContainer.vue -->
<template>
  <div class="sub-app-wrapper">
    <!-- Loading 状态 -->
    <div v-if="loading" class="sub-app-loading">
      <el-skeleton :rows="10" animated />
      <p>正在加载应用...</p>
    </div>

    <!-- 错误状态 -->
    <div v-else-if="error" class="sub-app-error">
      <el-result icon="error" title="应用加载失败" :sub-title="errorMessage">
        <template #extra>
          <el-button type="primary" @click="retry">重新加载</el-button>
          <el-button @click="goHome">返回首页</el-button>
        </template>
      </el-result>
    </div>

    <!-- 维护中状态 -->
    <div v-else-if="maintenance" class="sub-app-maintenance">
      <el-result
        icon="warning"
        title="应用维护中"
        sub-title="该应用正在维护升级，请稍后再试"
      >
        <template #extra>
          <el-button @click="goHome">返回首页</el-button>
        </template>
      </el-result>
    </div>

    <!-- 子应用挂载 -->
    <micro-app
      v-else
      :name="appId"
      :url="entryUrl"
      :baseroute="routePrefix"
      :data="appData"
      :keep-alive="keepAlive"
      @created="onCreated"
      @beforemount="onBeforeMount"
      @mounted="onMounted"
      @unmount="onUnmount"
      @error="onError"
      @datachange="onDataChange"
    />
  </div>
</template>

<script setup lang="ts">
const props = defineProps<{
  appId: string;
  entryUrl: string;
  routePrefix: string;
  keepAlive?: boolean;
}>();

const loading = ref(true);
const error = ref(false);
const errorMessage = ref('');
const maintenance = ref(false);

const appData = computed(() => ({
  token: useUserStore().token,
  permissions: usePermissionStore().permissions,
  theme: useSettingsStore().theme,
  userInfo: useUserStore().userInfo,
}));

// 加载超时处理
let loadingTimer: ReturnType<typeof setTimeout>;
onMounted(() => {
  loadingTimer = setTimeout(() => {
    if (loading.value) {
      loading.value = false;
      error.value = true;
      errorMessage.value = '加载超时，请检查网络连接';
    }
  }, 15000); // 15 秒超时
});

function onCreated() {
  loading.value = true;
  error.value = false;
}

function onBeforeMount() {
  // 资源加载完成
}

function onMounted() {
  loading.value = false;
  clearTimeout(loadingTimer);
}

function onUnmount() {
  // 子应用卸载
}

function onError(e: CustomEvent) {
  loading.value = false;
  error.value = true;
  errorMessage.value = e.detail?.error?.message || '未知错误';
  clearTimeout(loadingTimer);
}

function retry() {
  error.value = false;
  loading.value = true;
  // 强制重新渲染 micro-app 标签
  nextTick(() => {
    /* 触发重新挂载 */
  });
}

function goHome() {
  router.push('/dashboard');
}

onUnmounted(() => clearTimeout(loadingTimer));
</script>
```

### 2.2 生命周期时序

```
用户进入子应用路由
  │
  ├─ 1. 路由守卫检测子应用 → 加载菜单 → 渲染 SubAppContainer
  ├─ 2. SubAppContainer 展示 Loading
  ├─ 3. <micro-app> 标签创建 → created 事件
  ├─ 4. micro-app 加载子应用 HTML/JS/CSS 资源
  │      ├─ 成功 → beforemount → 渲染子应用 → mounted
  │      │         └─ SubAppContainer 隐藏 Loading
  │      └─ 失败 → error 事件
  │               └─ SubAppContainer 展示错误页
  ├─ 5. 用户切走（keep-alive 模式）
  │      └─ 子应用冻结，DOM 保留但不活跃
  ├─ 6. 用户切回
  │      └─ 子应用恢复，触发 activated 相关逻辑
  └─ 7. 子应用卸载 → unmount 事件
         └─ 子应用清理副作用
```

## 4. 沙箱配置

### 4.1 JS 沙箱

micro-app 默认启用 Proxy 沙箱：

```
JS 沙箱工作机制（Proxy）
  │
  ├─ 子应用运行时，window 对象被 Proxy 代理
  ├─ 子应用对 window 的读写操作被拦截到独立的沙箱环境
  ├─ 子应用间的全局变量互不干扰
  ├─ 子应用卸载时，沙箱内的全局状态自动清除
  └─ 特殊情况：某些第三方库需要访问真实 window
         └─ 使用 rawWindow: window.rawWindow 或 escape 属性配置白名单
```

**全局变量逃逸配置**（仅在必要时使用）：

```typescript
microApp.start({
  plugins: {
    global: [
      {
        escapeProperties: ['__SOME_GLOBAL_VAR__'], // 允许逃逸的全局变量
      },
    ],
  },
});
```

### 4.2 CSS 隔离

```
CSS 隔离模式选择
  │
  ├─ 方式一：Shadow DOM（默认，隔离最强）
  │      ├─ <micro-app> 内部渲染在 Shadow DOM 中
  │      ├─ 外部样式完全无法影响子应用
  │      ├─ 子应用样式完全无法影响外部
  │      └─ 主题 CSS 变量通过 :host 穿透
  │
  └─ 方式二：scopecss（可选，隔离较弱但兼容性好）
         ├─ <micro-app name="project" shadowDOM="false" />
         ├─ micro-app 自动为子应用的样式添加前缀
         └─ 主应用样式可以直接影响子应用（主题统一更容易）
```

**推荐策略**：默认使用 Shadow DOM，如果子应用需要继承主应用主题样式且 CSS 变量穿透不够用，可降级为 scopecss。

### 4.3 主题穿透

```css
/* 主应用定义 CSS 变量 */
:root {
  --primary-color: #1890ff;
  --bg-color: #ffffff;
  --text-color: #333333;
  --border-radius: 4px;
}

/* Shadow DOM 模式下，子应用通过 CSS 变量继承主题 */
/* CSS 变量天然穿透 Shadow DOM 边界 */
```

子应用使用 CSS 变量：

```css
.button-primary {
  background-color: var(--primary-color);
  border-radius: var(--border-radius);
}
```

### 4.4 存储隔离

```
localStorage / sessionStorage 隔离策略
  │
  ├─ micro-app 默认对子应用的存储操作添加应用名前缀
  │      ├─ 子应用调用 localStorage.setItem('token', '...')
  │      └─ 实际存储为 localStorage.setItem('micro-app-project-token', '...')
  ├─ 子应用卸载时，沙箱内的存储状态不自动清除（需要时手动清除）
  └─ 子应用间的存储数据互不干扰
```

## 5. 异常处理

### 5.1 错误边界

```typescript
// 全局错误处理
microApp.start({
  lifeCycles: {
    error(e) {
      const { name, error } = e.detail;
      // 上报错误
      reportAppError({
        appId: name,
        error: error?.message || 'Unknown error',
        stack: error?.stack,
        timestamp: Date.now(),
      });
    },
  },
});
```

### 5.2 子应用运行时异常捕获

```typescript
// 子应用内部异常不应导致主应用崩溃
// micro-app 已做隔离，以下为子应用自身的兜底

// Vue 子应用
app.config.errorHandler = (err, vm, info) => {
  console.error('[sub-app error]', err)
  window.microApp?.dispatch({
    type: 'app-error',
    error: { message: err.message, info }
  })
}

// React 子应用
class ErrorBoundary extends React.Component {
  componentDidCatch(error, errorInfo) {
    window.microApp?.dispatch({
      type: 'app-error',
      error: { message: error.message, stack: error.stack }
    })
  }
  render() {
    if (this.state.hasError) {
      return <div>子应用渲染异常</div>
    }
    return this.props.children
  }
}
```

### 5.3 资源加载失败重试

```
加载失败处理
  │
  ├─ 首次加载失败 → 展示错误页 + 重试按钮
  ├─ 用户点击重试 → 销毁 <micro-app> → 重新创建
  ├─ 自动重试策略（可选）
  │      ├─ 首次失败后 3 秒自动重试
  │      ├─ 第二次失败后 10 秒自动重试
  │      └─ 第三次失败后停止，展示错误页
  └─ 重试时展示 Loading 状态
```

## 6. keep-alive 管理

### 6.1 配置

```vue
<!-- keep-alive 模式 -->
<micro-app name="project" url="..." keep-alive />

<!-- 非 keep-alive 模式（每次重新加载） -->
<micro-app name="project" url="..." />
```

### 6.2 与标签页系统集成

```typescript
// 关闭标签页时卸载子应用
function closeAppTab(appId: string) {
  const container = document.querySelector(`micro-app[name="${appId}"]`);
  if (container) {
    // micro-app keep-alive 模式下，移除 DOM 会触发卸载
    container.parentNode?.removeChild(container);
  }
}
```

### 6.3 内存管理

```
keep-alive 子应用内存管理
  │
  ├─ keep-alive 的子应用 DOM 保留在内存中
  ├─ 同时激活的 keep-alive 子应用数量建议 ≤ 5
  ├─ 超过限制时，按 LRU 策略卸载最久未访问的子应用
  └─ 退出登录时卸载所有子应用
```

## 7. 编码规范

### 7.1 子应用入口适配

```typescript
// Vue 3 子应用入口 main.ts
let app: App | null = null;

function mount() {
  app = createApp(App);
  const base = window.__MICRO_APP_BASE_ROUTE__ || '/';
  const router = createRouter({
    history: createWebHistory(base),
    routes,
  });
  app.use(router);
  app.mount('#app');
}

function unmount() {
  app?.unmount();
  app = null;
}

// 微前端环境
if (window.__MICRO_APP_ENVIRONMENT__) {
  mount();
  window.addEventListener('unmount', unmount);
} else {
  // 独立运行
  mount();
}
```

### 7.2 注意事项

| 注意项    | 说明                                                            |
| --------- | --------------------------------------------------------------- |
| 路由 base | 子应用的 router base 必须使用 `window.__MICRO_APP_BASE_ROUTE__` |
| 资源路径  | 子应用内的静态资源使用绝对路径或 publicPath                     |
| 全局事件  | 子应用绑定的 window 事件需在 unmount 时清除                     |
| 定时器    | 子应用创建的 setInterval/setTimeout 需在 unmount 时清除         |
| WebSocket | 子应用建立的 WebSocket 连接需在 unmount 时断开                  |
| body 操作 | 避免直接操作 document.body，使用子应用容器内的 DOM              |

## 8. 安全要点

| 项目         | 要求                                                      |
| ------------ | --------------------------------------------------------- |
| 沙箱不可绕过 | 子应用不应通过 rawWindow 绕过沙箱限制（除非必要且经审核） |
| 全局变量逃逸 | escapeProperties 白名单需严格控制，每项需说明理由         |
| 异常上报     | 子应用异常需上报到统一留痕与埋点通道                      |
| 内存泄漏     | 子应用 unmount 时必须清理所有副作用                       |
| 资源清理     | keep-alive 子应用需关注内存占用，设置上限                 |
