# 07-04 公共依赖与接入规范 — 编码规范（Spec）

## 1. 公共依赖共享策略

### 1.1 共享方案

```
依赖共享层次
  │
  ├─ 第一层：externals + CDN（同技术栈共享）
  │      ├─ 将 Vue / React / Axios 等通过 externals 配置排除
  │      ├─ 主应用通过 CDN（或内网静态资源服务器）加载
  │      ├─ micro-app 通过 global 属性将全局库共享给子应用
  │      └─ 适用条件：子应用与主应用使用相同版本的框架
  │
  ├─ 第二层：npm 包（门户 SDK）
  │      ├─ 门户提供 @portal/micro-sdk 包
  │      ├─ 包含通信工具、权限 SDK、主题适配、HTTP 封装
  │      └─ 子应用通过 npm install 引入
  │
  └─ 第三层：子应用自包含（不同技术栈）
         ├─ 技术栈不同的子应用（如 React 子应用接入 Vue 门户）
         ├─ 自行打包所有依赖
         └─ micro-app 沙箱保证不冲突
```

### 1.2 CDN 共享配置

```html
<!-- 主应用 index.html -->

<script src="/cdn/vue-router@4.3.0/vue-router.global.prod.js"></script>

```


### 1.3 版本管理策略

| 策略         | 说明                                                   |
| ------------ | ------------------------------------------------------ |
| 主版本锁定   | 共享库锁定主版本号（如 Vue 3.x），子应用不可使用 Vue 2 |
| 次版本兼容   | 次版本号由门户统一升级，子应用无需感知                 |
| 独立打包兜底 | 如果子应用必须使用不同版本，不使用 externals，自行打包 |
| 版本矩阵     | 维护共享库版本矩阵文档，明确支持的版本范围             |

## 2. 门户 SDK 包

### 2.1 包清单

| 包名 | 说明 |
| --- | --- |
| `@portal/micro-sdk` | 微前端通信、生命周期、环境判断 |
| `@portal/permission-sdk` | 权限指令、组件、Hook |
| `@portal/http-client` | 统一 HTTP 请求封装（拦截器、Token 注入、错误处理） |
| `@portal/theme-sdk` | 主题变量、暗色模式适配工具 |
| `@portal/tracking-sdk` | 平台通用留痕与埋点上报（操作留痕、异常上报、访问埋点） |

### 2.2 @portal/http-client

```typescript
// 统一请求封装，子应用使用此客户端发起 API 请求
import { createHttpClient } from '@portal/http-client';

const http = createHttpClient({
  baseURL: '/api',
  getToken: () => {
    // 微前端环境：从主应用获取 Token
    if (window.__MICRO_APP_ENVIRONMENT__) {
      return window.microApp.getData()?.token;
    }
    // 独立运行：从本地存储获取
    return localStorage.getItem('token');
  },
  onUnauthorized: () => {
    // 微前端环境：通知主应用登出
    if (window.__MICRO_APP_ENVIRONMENT__) {
      window.microApp.dispatch({ type: 'logout' });
    } else {
      router.push('/login');
    }
  },
  onError: (error) => {
    ElMessage.error(error.message || '请求失败');
  },
});
```

**内置拦截器**：

```
请求拦截器
  ├─ 注入 Authorization: Bearer {token}
  ├─ 注入 X-App-Id: {appId}（标识请求来源子应用）
  └─ 注入 X-Request-Id: {uuid}（请求追踪）

响应拦截器
  ├─ 401 → 调用 onUnauthorized
  ├─ 403 → 提示无权限
  ├─ 500 → 提示服务器错误
  └─ 业务错误码 → 提取 message 展示
```

### 2.3 @portal/tracking-sdk

```typescript
import { createTracker } from '@portal/tracking-sdk';

const tracker = createTracker({
  appId: 'project-management',
  endpoint: '/api/tracking/report',
```

## 3. 子应用接入清单

### 3.1 接入步骤

```
子应用接入全流程
  │
  ├─ 步骤一：基础配置
  │      ├─ 安装 @portal/micro-sdk
  │      ├─ 配置 CORS（开发服务器 + 生产部署）
  │      ├─ 路由使用 history 模式，base 读取 window.__MICRO_APP_BASE_ROUTE__
  │      └─ 入口文件判断 window.__MICRO_APP_ENVIRONMENT__
  │
  ├─ 步骤二：生命周期适配
  │      ├─ 监听 unmount 事件，清理副作用
  │      ├─ 创建 / 销毁应用实例
  │      └─ 确保独立运行和微前端运行均正常
  │
  ├─ 步骤三：通信集成
  │      ├─ 使用 @portal/micro-sdk 接收 Token 和权限
  │      ├─ 使用 @portal/http-client 发起 API 请求
  │      ├─ 路由变更时通知主应用
  │      └─ 使用 @portal/permission-sdk 做权限控制
  │
  ├─ 步骤四：门户注册
  │      ├─ 管理员在门户注册子应用（应用标识、入口地址、路由前缀）
  │      ├─ 注册子应用菜单（POST /permissions/batch-register）
  │      ├─ 创建子应用角色并分配权限
  │      └─ 将角色分配给用户
  │
  └─ 步骤五：验证测试
         ├─ 使用接入验证工具检查
         ├─ 在门户中访问子应用，验证功能
         ├─ 验证独立运行模式
         └─ 验证权限控制
```

### 3.2 CORS 配置

**开发环境（Vite）**：

```typescript
// vite.config.ts
export default defineConfig({
  server: {
    port: 8081,
    cors: true,
    headers: {
      'Access-Control-Allow-Origin': '*',
    },
  },
});
```

**生产环境（Nginx）**：

```nginx
server {
    listen 80;
    server_name project.internal.com;

    location / {
        add_header Access-Control-Allow-Origin *;
        add_header Access-Control-Allow-Methods 'GET, POST, OPTIONS';
        add_header Access-Control-Allow-Headers 'DNT,X-Mx-ReqToken,Keep-Alive,User-Agent,X-Requested-With,If-Modified-Since,Cache-Control,Content-Type,Authorization';

        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }
}
```

### 3.3 子应用入口模板（Vue 3）

```typescript
// main.ts
import { createApp, type App as VueApp } from 'vue';
import { createRouter, createWebHistory } from 'vue-router';
import { setupPortalIntegration } from '@portal/micro-sdk';
import RootApp from './App.vue';
import { routes } from './router';

let app: VueApp | null = null;

function mount() {
  app = createApp(RootApp);

  const base = window.__MICRO_APP_BASE_ROUTE__ || '/';
  const router = createRouter({
    history: createWebHistory(base),
    routes,
  });

  app.use(router);

  // 微前端环境下集成门户能力
  if (window.__MICRO_APP_ENVIRONMENT__) {
    setupPortalIntegration(app, router);
  }

  app.mount('#app');
}

function unmount() {
  app?.unmount();
  app = null;
}

mount();

if (window.__MICRO_APP_ENVIRONMENT__) {
  window.addEventListener('unmount', unmount);
}
```

### 3.4 子应用入口模板（React 18）

```tsx
// index.tsx
import React from 'react';
import { createRoot, Root } from 'react-dom/client';
import { BrowserRouter } from 'react-router-dom';
import App from './App';

let root: Root | null = null;

function mount() {
  const base = (window as any).__MICRO_APP_BASE_ROUTE__ || '/';
  const container = document.getElementById('root')!;
  root = createRoot(container);
  root.render(
    <BrowserRouter basename={base}>
      <App />
    </BrowserRouter>,
  );
}

function unmount() {
  root?.unmount();
  root = null;
}

mount();

if ((window as any).__MICRO_APP_ENVIRONMENT__) {
  window.addEventListener('unmount', unmount);
}
```

## 7. 编码规范

### 7.1 子应用开发规范

| 规范               | 说明                                                |
| ------------------ | --------------------------------------------------- |
| 路由 base          | 必须使用 `window.__MICRO_APP_BASE_ROUTE__`          |
| 静态资源           | 使用绝对路径或正确配置 publicPath                   |
| 全局状态           | 微前端环境下使用主应用下发的 Token/权限，不自行管理 |
| HTTP 请求          | 使用 @portal/http-client，自动注入 Token            |
| 权限控制           | 使用 @portal/permission-sdk                         |
| 通用留痕与埋点上报 | 使用 @portal/tracking-sdk                           |
| unmount 清理       | 清理定时器、WebSocket、事件监听                     |
| 独立运行           | 确保脱离门户可独立运行和开发                        |

### 7.2 不允许的行为

| 行为                              | 原因                   |
| --------------------------------- | ---------------------- |
| 直接操作 `document.body`          | 破坏主应用布局         |
| 覆盖 `window.location`            | 影响主应用路由         |
| 使用 `document.cookie` 管理 Token | Token 由主应用统一管理 |
| 全局注册同名组件                  | 可能与其他子应用冲突   |
| 修改 `document.title`             | 通过事件通知主应用修改 |

## 8. 安全要点

| 项目      | 要求                                     |
| --------- | ---------------------------------------- |
| CORS 限制 | 生产环境 CORS 不使用 `*`，限定为门户域名 |
| 入口地址  | 仅允许内网地址作为子应用入口             |
| SDK 版本  | 子应用使用的 SDK 版本需与门户兼容        |
| 代码审查  | 子应用接入前需通过接入验证工具检查       |
| 依赖安全  | 共享 CDN 库需来自可信源（内网镜像）      |
