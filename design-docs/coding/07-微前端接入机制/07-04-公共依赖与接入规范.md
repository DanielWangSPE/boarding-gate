# 07-04 公共依赖与接入规范

## 1. 功能说明

本模块定义子应用接入门户的技术规范、公共依赖共享策略，以及门户为子应用提供的 SDK 和脚手架工具。确保子应用能够高效、规范地接入门户平台。

## 2. 功能清单

### 2.1 公共依赖共享

- 公共基础库共享（如 Vue、React、Axios、Lodash 等）
- 共享库的版本管理和兼容性策略
- micro-app 支持通过 `global` 属性将全局资源共享给子应用
- 同技术栈子应用可通过 externals + CDN 共享公共库以减少加载体积

### 2.2 公共组件/工具库

- 提供统一的 UI 组件库（或规定统一的组件库规范）
- 提供公共工具函数库（请求封装、权限判断、格式化工具等）
- 以 npm 包形式发布，子应用通过依赖引用

### 2.3 子应用接入清单

- 子应用需配置允许跨域访问（CORS），支持 micro-app 的资源加载
- 子应用的路由需使用 history 模式，并基于门户分配的路由前缀设置 `base`
- 子应用需在入口文件中判断 `window.__MICRO_APP_ENVIRONMENT__`，区分微前端/独立运行环境
- 子应用需监听 `unmount` 事件，执行清理操作
- 子应用需通过门户接口注册自己的菜单和权限资源
- 子应用**无需**导出生命周期函数或修改构建输出格式

### 2.4 接入文档与脚手架

- 提供详细的子应用接入文档
- 提供不同技术栈的接入脚手架模板（Vue、React 等）
- 提供接入验证工具（检查子应用是否符合接入规范）
- 提供子应用本地联调方案（子应用可脱离主应用独立运行和开发）

## 3. 业务规则

| 编号  | 规则      | 说明                                           |
| ----- | --------- | ---------------------------------------------- |
| BR-01 | 独立运行  | 子应用应能脱离主应用独立运行（用于开发和调试） |
| BR-02 | 版本兼容  | 子应用版本更新不应导致主应用框架需要同步更新   |
| BR-03 | CORS 必须 | 子应用必须配置跨域，否则无法在门户中加载       |
| BR-04 | SDK 统一  | 子应用通过门户提供的 SDK 使用权限、通信等能力  |
