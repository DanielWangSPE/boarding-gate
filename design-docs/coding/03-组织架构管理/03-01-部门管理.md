# 03-01 部门管理

## 1. 功能说明

本模块负责公司组织树的节点维护，包括部门信息管理、树形结构操作（新增/编辑/删除/移动）和部门状态管理。

## 2. 功能清单

### 2.1 部门信息

- 部门名称
- 部门编码（唯一标识）
- 上级部门
- 部门负责人（关联用户）
- 部门排序号
- 部门状态（正常 / 停用）
- 部门描述/备注

### 2.2 部门树操作

- 以树形结构展示公司组织架构
- 支持新增、编辑、删除部门
- 支持拖拽调整部门层级和排序
- 停用部门时需处理下属人员和子部门

## 3. 业务规则

| 编号  | 规则         | 说明                                       | 配置项 |
| ----- | ------------ | ------------------------------------------ | ------ |
| BR-01 | 根部门保护   | 根部门（公司）不可删除、不可停用、不可移动 | —      |
| BR-02 | 部门编码唯一 | 部门编码全局唯一，创建后不可修改           | —      |
| BR-03 | 删除限制     | 部门下存在用户或子部门时不可直接删除       | —      |
| BR-04 | 停用级联     | 停用部门时，其下所有子部门同步停用         | —      |
| BR-05 | 同级名称唯一 | 同一父部门下部门名称不可重复               | —      |
| BR-06 | 禁止循环     | 移动部门时不可将部门移到自身或其下级部门下 | —      |

## 4. 接口要求

| 接口                     | 方法   | 说明                      |
| ------------------------ | ------ | ------------------------- |
| /departments             | GET    | 获取部门树                |
| /departments             | POST   | 创建部门                  |
| /departments/{id}        | GET    | 获取部门详情              |
| /departments/{id}        | PUT    | 更新部门                  |
| /departments/{id}        | DELETE | 删除部门                  |
| /departments/{id}/status | PUT    | 启用/停用部门             |
| /departments/{id}/move   | PUT    | 移动部门（调整层级/排序） |

## 5. 页面清单

| 页面         | 说明                      |
| ------------ | ------------------------- |

| 部门编辑弹窗 | 新增/编辑部门信息         |
