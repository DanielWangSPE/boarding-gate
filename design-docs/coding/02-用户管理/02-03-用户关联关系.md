# 02-03 用户关联关系

## 1. 功能说明

本模块管理用户与组织部门、角色之间的关联关系，是权限体系和数据权限的基础。

## 2. 功能清单

### 2.1 用户-组织关系

- 用户必须归属一个组织节点
- 用户调动时支持变更所属部门

### 2.2 用户-角色关系

- 用户可被分配一个或多个角色
- 角色来源：系统全局角色 + 子应用角色
- 支持批量分配角色
- 角色变更后权限实时生效

## 3. 业务规则

| 编号  | 规则         | 说明                                   |
| ----- | ------------ | -------------------------------------- |
| BR-01 | 必须有部门   | 用户必须归属一个组织节点               |
| BR-02 | 部门有效性   | 不可将用户关联到已停用的部门           |
| BR-03 | 角色有效性   | 不可将已停用的角色分配给用户           |
| BR-04 | 权限即时生效 | 角色分配变更后，用户权限缓存需主动刷新 |

## 4. 接口要求

| 接口                    | 方法 | 说明                     |
| ----------------------- | ---- | ------------------------ |
| /users/{id}/departments | GET  | 查询用户所属组织节点     |
| /users/{id}/departments | PUT  | 设置用户所属组织节点     |
| /users/{id}/roles       | GET  | 查询用户角色列表         |
| /users/{id}/roles       | PUT  | 分配用户角色（全量替换） |

## 5. 页面清单

| 页面                    | 说明                                              |
| ----------------------- | ------------------------------------------------- |
| 用户详情页-关联关系 Tab | 在用户详情页中以 Tab 形式展示和编辑部门、角色关联 |
