# 02-02 用户生命周期

## 1. 功能说明

本模块管理用户从创建、启用、停用到注销的完整生命周期，包括用户创建、状态流转和账号注销/恢复。

## 2. 功能清单

### 2.1 用户创建

- 管理员手动创建用户
- 创建时设置初始密码（系统生成或手动指定）
- 创建时可关联组织、角色

### 2.2 用户状态管理

- 用户状态：正常 / 停用 / 锁定
- 管理员可手动启用/停用用户
- 停用后用户无法登录，已有会话立即失效
- 锁定状态由系统自动管理（如密码错误次数超限），支持手动解锁

### 2.3 用户注销

- 支持用户账号注销（软删除，保留数据记录）
- 注销后释放用户名，但保留历史操作记录
- 注销操作需二次确认
- 管理员可在一定期限内恢复已注销账号

## 3. 状态流转图

```
                   创建
                    │
                    ▼
     ┌──────────► 正常 ◄──────────┐
     │              │              │
     │         停用 / 锁定         │
     │              │              │
     │              ▼              │
     │          停用/锁定 ─────────┘
     │              │        (启用/解锁)
     │              │
     │           注销
     │              │
     │              ▼
     └────────── 已注销
          (恢复，限期内)
```

## 4. 业务规则

| 编号 | 规则 | 说明 |
| --- | --- | --- |
| BR-01 | 停用即踢 | 用户被停用后，所有活跃会话立即失效 |
| BR-02 | 超级管理员保护 | 超级管理员账号不可被停用或删除 |
| BR-03 | 自我保护 | 用户不可停用或删除自己的账号 |
| BR-04 | 初始密码 | 系统生成的初始密码须符合密码复杂度策略 |
| BR-05 | 注销期限 | 注销后可恢复期限为配置项（默认 30 天），过期后不可恢复 |
| BR-06 | 注销释放 | 注销后用户名释放，可被新用户使用 |

## 5. 接口要求

| 接口                | 方法   | 说明               |
| ------------------- | ------ | ------------------ |
| /users              | POST   | 创建用户           |
| /users/{id}/status  | PUT    | 启用/停用用户      |
| /users/{id}/unlock  | POST   | 解锁用户           |
| /users/{id}         | DELETE | 注销用户（软删除） |
| /users/{id}/restore | POST   | 恢复已注销用户     |
