# 02-01 用户信息管理

## 1. 功能说明

本模块负责定义和维护用户的基本信息与扩展信息，是用户管理的数据基础。

> 用户创建、启用/停用、解锁、注销/恢复等生命周期流转能力由 `02-02 用户生命周期` 模块负责；本模块聚焦于信息结构定义和信息维护。

## 2. 功能清单

### 2.1 用户基本信息

- 用户名（登录账号，全局唯一）
- 姓名（真实姓名）
- 手机号（普通信息字段，非必填）
- 邮箱（普通信息字段，非必填）
- 头像（支持上传自定义头像）
- 性别
- 工号（员工编号，可选）
- 用户类型（内部用户 / 外部用户 / 系统账号）

### 2.2 用户扩展信息

- 支持自定义扩展字段（由管理员配置）
- 扩展字段支持多种数据类型（文本、数值、日期、下拉选择、多选等）
- 扩展字段可配置是否必填、是否在列表中展示

## 3. 业务规则

| 编号  | 规则       | 说明                                                   |
| ----- | ---------- | ------------------------------------------------------ |
| BR-01 | 用户名唯一 | 用户名全局唯一，不可重复（注销释放后可复用）           |
| BR-02 | 手机号唯一 | 手机号非必填，但填写时同一用户类型下不可重复           |
| BR-03 | 邮箱唯一   | 邮箱非必填，但填写时同一用户类型下不可重复             |
| BR-04 | 用户名规范 | 仅允许字母、数字、下划线，长度 4-30 位，创建后不可修改 |
| BR-05 | 头像限制   | 仅支持 jpg/png/gif 格式，大小不超过 2MB                |

## 4. 接口要求

| 接口               | 方法   | 说明                 |
| ------------------ | ------ | -------------------- |
| /users/{id}        | GET    | 获取用户详情         |
| /users/{id}        | PUT    | 更新用户信息         |
| /users/current     | GET    | 获取当前登录用户信息 |
| /users/{id}/avatar | POST   | 上传用户头像         |
| /user-fields       | GET    | 获取扩展字段定义列表 |
| /user-fields       | POST   | 创建扩展字段         |
| /user-fields/{id}  | PUT    | 更新扩展字段定义     |
| /user-fields/{id}  | DELETE | 删除扩展字段定义     |

## 5. 页面清单

| 页面            | 说明                                     |
| --------------- | ---------------------------------------- |
| 用户详情/编辑页 | 查看/编辑用户基本信息和扩展信息          |
| 扩展字段管理页  | 管理员配置自定义扩展字段（系统管理入口） |
