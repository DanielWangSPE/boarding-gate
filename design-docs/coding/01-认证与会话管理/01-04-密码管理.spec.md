# 01-04 密码管理 — 编码规范（Spec）

## 1. 数据模型

### 1.1 用户表密码相关字段

```sql
ALTER TABLE sys_user ADD COLUMN password_hash      VARCHAR(128) NOT NULL COMMENT '密码哈希（BCrypt）';
ALTER TABLE sys_user ADD COLUMN password_changed_at DATETIME     NULL     COMMENT '最近一次修改密码时间';
ALTER TABLE sys_user ADD COLUMN force_change_pwd    TINYINT(1)   NOT NULL DEFAULT 1 COMMENT '是否需要强制修改密码：1-是 0-否';
```

### 1.2 密码历史表

```sql
CREATE TABLE sys_password_history (
  id            BIGINT       PRIMARY KEY AUTO_INCREMENT,
  user_id       BIGINT       NOT NULL COMMENT '用户 ID',
  password_hash VARCHAR(128) NOT NULL COMMENT '历史密码哈希（BCrypt）',
  created_at    DATETIME     NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '记录时间',
  INDEX idx_user_id (user_id)
) COMMENT '密码历史记录表';
```

## 2. 接口规范

### 2.1 用户修改密码

```
POST /auth/password/change
Authorization: Bearer {accessToken}
Content-Type: application/json
```

**Request Body**

```json
{
  "oldPassword": "string（RSA 加密密文）",
  "newPassword": "string（RSA 加密密文）",
  "confirmPassword": "string（RSA 加密密文）"
}
```

**Response 200**

```json
{
  "code": 200,
  "message": "密码修改成功，请重新登录"
}
```

**Response 错误码**

| 业务 code | 说明                      |
| --------- | ------------------------- |
| A0250     | 原密码错误                |
| A0251     | 新密码不符合复杂度要求    |
| A0252     | 新密码与最近 N 次密码重复 |
| A0253     | 新密码与确认密码不一致    |
| A0254     | 新密码不能与原密码相同    |

### 2.2 管理员重置用户密码

```
POST /admin/users/{userId}/reset-password
Authorization: Bearer {accessToken}
Content-Type: application/json
```

**Request Body**

```json
{
  "newPassword": "string（RSA 加密密文，可选。为空则系统自动生成）"
}
```

**Response 200**

```json
{
  "code": 200,
  "data": {
    "generatedPassword": "string（仅自动生成时返回，供管理员告知用户）"
  },
  "message": "密码重置成功"
}
```

**实现要点**：

- 需要 `admin:user:reset-password` 权限
- 重置后将用户 `force_change_pwd` 设为 1
- 自动生成的密码需符合当前密码复杂度策略
- 重置成功后调用会话治理能力，使目标用户当前所有会话失效
- 响应中的 `generatedPassword` 为明文，仅在响应中出现一次，不做持久化

## 3. 处理流程

### 3.1 用户修改密码

```
用户提交修改密码表单
  │
  ├─ 1. 前端校验
  │      ├─ 新密码与确认密码一致
  │      ├─ 新密码符合复杂度（前端预校验，后端最终校验）
  │      └─ RSA 加密所有密码字段
  │
  ├─ 2. 后端处理
  │      ├─ RSA 解密获取明文
  │      ├─ BCrypt 校验原密码 → 失败返回 A0250
  │      ├─ 校验新密码 ≠ 原密码 → 相同返回 A0254
  │      ├─ 校验密码复杂度 → 不通过返回 A0251
  │      ├─ 查询 sys_password_history 最近 N 条
  │      │    └─ 逐条 BCrypt matches → 命中返回 A0252
  │      ├─ BCrypt 哈希新密码
  │      ├─ 更新 sys_user.password_hash
  │      ├─ 更新 sys_user.password_changed_at
  │      ├─ 更新 sys_user.force_change_pwd = 0
  │      ├─ 写入 sys_password_history
  │      ├─ 清理超出 historyCount 的历史记录
  │      ├─ 将当前 Access Token 加入黑名单
  │      ├─ 删除当前会话记录并清除 Refresh Token Cookie
  │      └─ 记录操作留痕（异步）
  │
  └─ 3. 前端收到成功响应
         ├─ 清除本地登录态
         └─ 跳转登录页重新登录
```

### 3.2 强制改密判断（登录流程内嵌）

```
登录验证通过后
  │
  ├─ 检查 force_change_pwd == 1 ?
  │      └─ 是 → 返回 Token + forceChangePassword: true
  │
  ├─ 检查 password_changed_at + maxAgeDays < NOW() ?
  │      └─ 是（密码已过期）→ 返回 Token + forceChangePassword: true
  │
  └─ 否 → 正常登录
```

前端收到 `forceChangePassword: true` 后强制跳转至改密页面，改密完成前拦截所有其他路由。

## 4. 编码规范

### 4.1 密码复杂度校验器

后端实现统一的密码复杂度校验工具类，从配置中心动态读取规则：

```java
// 伪代码
public class PasswordPolicyValidator {
    public ValidationResult validate(String plainPassword) {
        // 从配置中心读取策略
        // 逐条检查：长度、大写、小写、数字、特殊字符
        // 返回校验结果（通过 / 不通过 + 具体原因）
    }
}
```

前端同样实现一份校验逻辑（从配置接口获取规则），用于表单实时提示，但**后端校验为最终权威**。

### 4.2 密码哈希

- 统一使用 BCrypt，cost factor ≥ 10
- 禁止使用 MD5、SHA-1、SHA-256 等快速哈希算法
- 禁止自行拼接盐值，BCrypt 内置盐值管理

### 4.3 密码历史比对

- 历史密码存储的是 BCrypt 哈希，无法反向解密
- 比对时需要用新密码明文逐条与历史哈希做 `BCrypt.matches`
- 当 `historyCount` 较大时会有性能开销，建议上限不超过 24
- 历史记录按 `created_at DESC` 排序，仅取前 N 条比对

### 4.4 随机密码生成

系统生成的随机密码需符合当前密码复杂度策略：

```
规则：
  - 长度 = max(配置最小长度, 12)
  - 至少包含 1 个大写字母
  - 至少包含 1 个小写字母
  - 至少包含 1 个数字
  - 如配置要求特殊字符，至少包含 1 个
  - 剩余位数随机填充上述字符集
  - 最终打乱顺序
```

### 4.5 前端规范

- 强制改密页面为全屏独立页面，不展示侧边栏和导航菜单
- 强制改密状态下，前端路由守卫拦截所有非改密页面的访问
- 密码输入框旁实时展示复杂度检查结果（逐条打勾）
- 新密码和确认密码一致性实时校验
- 密码强度指示条（弱 / 中 / 强）

### 4.6 操作留痕

- 用户修改密码：记录操作留痕（操作人 = 用户自身）
- 管理员重置密码：记录操作留痕（操作人 = 管理员，目标 = 被重置用户）
- 日志中禁止记录密码明文或密文

### 4.7 会话处理

- 用户主动修改密码时，仅使当前会话失效，其他已登录会话不主动下线
- 管理员重置密码时，使目标用户的所有会话失效
- 会话失效通过“删除会话记录 + Access Token 黑名单 + 清除 Refresh Token Cookie/校验失效”组合实现

## 5. 安全要点

| 项目             | 要求                                                   |
| ---------------- | ------------------------------------------------------ |
| 密码传输         | RSA 加密 + HTTPS                                       |
| 密码存储         | BCrypt（cost ≥ 10）                                    |
| 密码历史         | BCrypt 哈希存储，不可逆                                |
| 用户改密后生效   | 当前会话立即失效，需重新登录；其他已登录会话不主动下线 |
| 管理员重置后生效 | 目标用户所有会话立即失效                               |
| 管理员重置       | 自动生成密码仅返回一次，不持久化明文                   |
| 错误信息         | 原密码错误时明确提示，不泄露其他信息                   |
