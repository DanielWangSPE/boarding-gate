# 01-04 密码管理

## 1. 功能说明

本模块负责密码策略配置与执行、密码修改、管理员重置密码等功能。由于系统为内网部署，密码找回功能不适用，遗忘密码时由管理员进行重置。

## 2. 功能清单

### 2.1 密码策略

- 密码复杂度规则可配置（最小长度、是否要求大小写、是否要求数字、是否要求特殊字符，均为配置项）
- 密码历史记录，防止重复使用最近 N 次的密码（历史次数为配置项）
- 密码有效期（有效天数为配置项，到期强制修改；设为 0 表示永不过期）
- 首次登录强制修改初始密码

### 2.2 密码修改

- 用户可在个人中心修改密码
- 个人中心仅提供页面入口，实际调用认证模块接口 `/auth/password/change`
- 修改密码需验证原密码
- 管理员可重置用户密码（重置为系统生成的随机密码或管理员指定的密码）

## 3. 业务规则

| 编号 | 规则 | 说明 | 配置项 |
| --- | --- | --- | --- |
| BR-01 | 密码复杂度 | 密码的复杂度要求 | `auth.password.minLength`（默认 8）<br>`auth.password.requireUppercase`（默认 true）<br>`auth.password.requireLowercase`（默认 true）<br>`auth.password.requireDigit`（默认 true）<br>`auth.password.requireSpecialChar`（默认 false） |
| BR-02 | 密码历史 | 不允许使用最近 N 次使用过的密码 | `auth.password.historyCount`（默认 1） |
| BR-03 | 密码有效期 | 密码到期强制修改，0 表示永不过期 | `auth.password.maxAgeDays`（默认 90 天） |
| BR-04 | 首次改密 | 使用初始密码首次登录时必须修改密码 | — |
| BR-05 | 改密生效 | 用户主动修改密码后，当前会话需重新登录；其他已登录会话不主动下线 | — |
| BR-06 | 重置标记 | 管理员重置密码后，标记用户为"需首次改密"状态 | — |
| BR-07 | 重置后会话失效 | 管理员重置密码后，目标用户当前所有会话立即失效 | — |

## 4. 接口要求

| 接口                                 | 方法 | 说明               |
| ------------------------------------ | ---- | ------------------ |
| /auth/password/change                | POST | 用户修改密码       |
| /admin/users/{userId}/reset-password | POST | 管理员重置用户密码 |

## 5. 页面清单

| 页面              | 说明                             |
| ----------------- | -------------------------------- |
| 强制改密页        | 首次登录或密码过期时强制修改密码 |
| 个人中心-修改密码 | 用户主动修改密码（个人中心模块） |
