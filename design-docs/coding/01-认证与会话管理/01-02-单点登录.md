# 01-02 单点登录（SSO）

## 1. 功能说明

本模块负责在门户平台与各子应用之间实现统一的身份认证共享。在当前内网部署场景下，用户在门户登录一次后访问各子应用无需重复登录，并由主应用统一维护当前会话的登录状态。

## 2. 功能清单

### 2.1 SSO 能力

- 用户在门户登录后，访问任意子应用无需再次登录
- 主应用向子应用下发当前会话的登录态和用户基础信息
- 子应用复用主应用提供的 Token 调用后端接口
- 子应用不再单独建设登录页和独立登录态

### 2.2 统一登出

- 用户在门户登出后，当前已加载的子应用同步清理本次会话状态
- 子应用主动触发登出时，由主应用统一执行当前会话登出
- 登出时清除当前会话相关的 Token 和运行时状态

## 3. 业务规则

| 编号 | 规则 | 说明 |
| --- | --- | --- |
| BR-01 | 登录即通行 | 用户在主应用登录后，子应用通过 Token 验证身份，无需二次登录 |
| BR-02 | 登出联动 | 主应用登出后，当前已加载子应用同步清理本次会话状态 |
| BR-03 | Token 透传 | 子应用通过主应用下发的 Token 调用后端 API，网关统一校验 |
| BR-04 | 子应用主动登出 | 子应用触发登出时，统一委托主应用处理当前会话登出 |

## 4. 接口要求

| 接口           | 方法 | 说明                                         |
| -------------- | ---- | -------------------------------------------- |
| /auth/logout   | POST | 当前会话统一登出                             |
| /auth/userinfo | GET  | 子应用获取当前登录用户基础信息（基于 Token） |

## 5. 页面清单

本模块无独立页面，SSO 逻辑内嵌于登录流程和子应用加载流程中。
