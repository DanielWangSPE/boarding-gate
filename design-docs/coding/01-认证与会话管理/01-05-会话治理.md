# 01-05 会话治理

## 1. 功能说明

本模块负责提供必要的会话失效治理能力，支撑用户停用/注销后的统一下线、管理员重置密码后的会话失效，以及账号异常场景下的安全处置。

## 2. 功能清单

### 2.1 会话失效治理

- 管理员重置密码后，目标用户全部会话立即失效
- 用户被停用或注销后，目标用户全部会话立即失效
- 微前端主应用在会话被动失效后，统一清理登录态并跳转登录页

## 3. 业务规则

| 编号 | 规则 | 说明 |
| --- | --- | --- |
| BR-01 | 会话失效策略 | 用户全部会话失效后，旧 Access Token 立即加入黑名单，Refresh Token 对应会话不可再续期 |
| BR-02 | 内部调用边界 | 用户停用、注销、管理员重置密码等场景通过认证模块统一执行会话失效，避免其他模块直接操作 Redis |

## 4. 接口要求

| 接口 | 方法 | 说明 |
| --- | --- | --- |
| /internal/auth/users/{userId}/invalidate-sessions | POST | 供内部模块调用，使指定用户全部会话失效 |

## 5. 页面清单

本模块无独立页面，能力由后台内部流程和登录态失效处理逻辑承接。
