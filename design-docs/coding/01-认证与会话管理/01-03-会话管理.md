# 01-03 会话管理

## 1. 功能说明

本模块负责用户会话的全生命周期管理，包括 Token 签发与续期、会话保活与超时处理。

## 2. 功能清单

### 2.1 Token 管理

- 采用 Access Token + Refresh Token 双 Token 机制
- Access Token 短期有效（有效时长为配置项，默认 30 分钟）
- Refresh Token 长期有效（有效时长为配置项，默认 7 天）
- 登录成功后仅返回 `Access Token`，`Refresh Token` 通过 `HttpOnly Cookie` 保存
- Token 无感刷新机制（Access Token 过期时自动使用 Refresh Token 换取新 Token）
- Token 黑名单机制（支持主动吊销 Token）

### 2.2 会话保活与超时

- 用户活跃状态下自动续期会话
- 用户无操作超时自动登出（超时时间为配置项）
- 支持前端通过心跳或活跃上报延迟更新会话活跃时间

## 3. 业务规则

| 编号 | 规则 | 说明 | 配置项 |
| --- | --- | --- | --- |
| BR-01 | Token 有效期 | Access Token 和 Refresh Token 的有效时长 | `auth.token.accessExpire`（默认 30 分钟）<br>`auth.token.refreshExpire`（默认 7 天） |
| BR-02 | 会话超时 | 用户无操作超时自动登出 | `auth.session.idleTimeout`（默认 60 分钟） |
| BR-03 | Token 黑名单 | 吊销的 Token 在剩余有效期内拒绝访问 | — |

## 4. 接口要求

| 接口                    | 方法 | 说明                 |
| ----------------------- | ---- | -------------------- |
| /auth/refresh           | POST | 刷新 Token           |
| /auth/session/heartbeat | POST | 上报当前会话活跃状态 |

## 5. 页面清单

本模块无独立页面，相关逻辑内嵌于登录态维护和路由守卫中。
