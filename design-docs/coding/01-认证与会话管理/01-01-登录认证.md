# 01-01 登录认证

## 1. 功能说明

本模块负责用户通过账号密码方式登录系统的全流程，包括表单校验、图形验证码、错误次数限制与账号锁定及登录后跳转。

> **当前范围**：仅包含账号密码登录，可按配置开启图形验证码。

## 2. 功能清单

### 2.1 账号密码登录

- 支持用户名 + 密码登录
- 已成功登录过的用户名支持在登录页选中回填
- 密码字段由用户手动输入，不允许浏览器或密码管理器自动回填
- 密码输入框支持显示/隐藏切换，前端建议使用普通文本输入框承载并自行控制掩码展示
- 密码传输前使用国产密码算法公钥加密
- 登录页面支持图形验证码（可配置是否开启，支持连续失败后强制开启，失败次数阈值为配置项）
- 密码错误次数限制（阈值为配置项），超出后锁定账号一定时间（锁定时长为配置项）
- 登录成功后返回 `Access Token`，并由服务端写入 `Refresh Token` 的 `HttpOnly Cookie`
- 登录成功后跳转至用户上次访问的页面或门户首页

## 3. 业务规则

| 编号 | 规则 | 说明 | 配置项 |
| --- | --- | --- | --- |
| BR-01 | 密码错误锁定 | 连续输入错误密码达到阈值后，账号锁定一定时间 | `auth.login.maxRetryCount`（默认 5） `auth.login.lockDuration`（默认 30 分钟） |
| BR-02 | 验证码策略 | 可全局开启/关闭；连续登录失败达到阈值后强制开启 | `auth.captcha.enabled`（默认 true） `auth.captcha.triggerAfterFails`（默认 3） |
| BR-03 | 账号状态校验 | 登录时需校验账号状态：停用、锁定状态不允许登录 | — |
| BR-04 | 登录跳转 | 登录成功后优先跳转至 `redirect` 参数指定页面，无则跳转门户首页 | — |
| BR-05 | 用户名回填 | 仅允许回填历史登录成功的用户名，不回填密码 | — |

## 4. 接口要求

| 接口               | 方法 | 说明                  |
| ------------------ | ---- | --------------------- |
| /auth/crypto/login | GET  | 获取登录 SM4 加密参数 |
| /auth/crypto/login | POST | 账号密码登录          |
| /auth/captcha      | GET  | 获取图形验证码        |
