# 统一门户平台 — 需求总览

## 1. 项目背景

公司内部存在多个业务系统，各系统之间在登录认证、用户管理、角色权限等公共能力上存在大量重复建设，导致维护成本高、用户体验不一致、安全策略难以统一。

为解决上述问题，规划建设 **统一门户平台（Boarding Gate）**，作为公司级的基础设施平台，承担以下核心职责：

- **统一入口**：为所有业务系统提供统一的访问入口和门户首页
- **统一认证**：集中管理用户登录登出、单点登录（SSO）和登录状态等
- **统一授权**：集中管理用户、角色、权限、数据权限等
- **统一接入**：通过微前端架构将各业务系统作为子应用无缝接入

## 2. 系统定位

| 维度     | 说明                                     |
| -------- | ---------------------------------------- |
| 系统类型 | 企业级统一门户 + 微前端主应用            |
| 核心用户 | 公司全体员工、系统管理员、子应用开发团队 |
| 技术架构 | 微前端架构（主应用 + N 个子应用）        |
| 部署模式 | 支持私有化部署                           |

## 3. 模块全景

```
统一门户平台
├── 01-认证与会话管理        # 账号密码登录、SSO、Token 会话、密码安全、必要的会话治理
├── 02-用户管理              # 用户全生命周期管理
├── 03-组织架构管理          # 组织树结构、用户组织归属
├── 04-角色管理              # 角色定义、权限关联、用户分配
├── 05-权限管理              # 功能权限、数据权限、统一鉴权
├── 06-菜单与导航管理        # 门户菜单展示、应用入口导航、权限过滤
├── 07-微前端接入机制        # 子应用注册与管理、生命周期、沙箱隔离、通信机制、接入规范
├── 08-系统管理              # 数据字典、参数配置、文件服务
└── 09-门户首页与个人中心    # 首页欢迎页、个人信息与安全设置
```

## 4. 各模块概述

### 4.1 认证与会话管理

负责用户身份认证与登录状态管理，包括账号密码登录、可选图形验证码、SSO 单点登录、Token 与会话生命周期管理、密码安全能力，以及必要的会话治理等。

### 4.2 用户管理

覆盖用户从创建到停用的完整生命周期，包括用户信息维护、账号状态管理以及用户与组织/角色的关联关系等。

### 4.3 组织架构管理

管理公司的组织树结构，维护各组织节点之间的父子关系，以及用户所属的组织节点。

### 4.4 角色管理

定义和管理系统中的角色信息，维护角色与权限、角色与用户的关联关系。

### 4.5 权限管理

建设统一的权限模型（RBAC + ABAC），涵盖功能权限（菜单、按钮）和数据权限（行级、列级），后端接口鉴权复用功能权限标识，支持跨子应用的权限管控。

### 4.6 菜单与导航管理

统一管理门户侧的菜单展示和导航结构，支持应用入口展示、菜单权限过滤、菜单排序、层级组织与导航联动。

### 4.7 微前端接入机制

定义子应用的接入标准与规范，包括子应用注册与管理、生命周期管理、JS/CSS 沙箱隔离、应用间通信、公共依赖共享与接入规范等。

### 4.8 系统管理

提供平台级的基础管理能力，包括数据字典、系统参数配置和文件服务等。

### 4.9 门户首页与个人中心

为用户提供统一的门户首页和个人中心。其中门户首页阶段性仅保留欢迎文案，个人中心承载个人信息维护和密码修改等基础能力。

## 5. 术语表

| 术语   | 说明                                                   |
| ------ | ------------------------------------------------------ |
| 主应用 | 统一门户平台本身，负责整体框架、路由调度和公共能力     |
| 子应用 | 接入门户的各业务系统，通过微前端机制在主应用框架内运行 |
| SSO    | Single Sign-On，单点登录                               |
| RBAC   | Role-Based Access Control，基于角色的访问控制          |
| ABAC   | Attribute-Based Access Control，基于属性的访问控制     |
| 沙箱   | 微前端中用于隔离子应用运行环境的机制                   |
