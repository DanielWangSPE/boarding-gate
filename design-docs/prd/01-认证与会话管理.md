# 01 - 认证与会话管理

## 1. 模块概述

认证与会话管理模块是统一门户平台的安全基石，负责账号密码登录、单点登录、Token 与会话生命周期、密码安全能力，以及必要的会话治理能力建设。所有子应用的认证需求统一由本模块承接，避免各系统重复建设登录体系。

> **部署环境**：内网部署，不涉及公网访问场景。
>
> **统一约定**：
>
> - 登录成功后由服务端返回 `Access Token`
> - `Refresh Token` 仅通过 `HttpOnly Cookie` 下发和续期，前端 JavaScript 不可直接读取
> - 个人中心中的密码修改页面入口归门户承载，认证能力与接口归本模块负责

## 3. 全局配置项汇总

| 配置项 | 说明 | 默认值 | 所属模块 |
| --- | --- | --- | --- |
| `auth.login.maxRetryCount` | 密码错误锁定阈值 | 5 次 | 登录认证 |
| `auth.login.lockDuration` | 账号锁定时长 | 30 分钟 | 登录认证 |
| `auth.captcha.enabled` | 是否开启图形验证码 | true | 登录认证 |
| `auth.captcha.triggerAfterFails` | 连续失败多少次后强制验证码 | 3 次 | 登录认证 |
| `auth.token.accessExpire` | Access Token 有效时长 | 30 分钟 | 会话管理 |
| `auth.token.refreshExpire` | Refresh Token 默认有效时长 | 7 天 | 会话管理 |
| `auth.session.idleTimeout` | 无操作超时时间 | 60 分钟 | 会话管理 |
| `auth.password.minLength` | 密码最小长度 | 8 | 密码管理 |
| `auth.password.requireUppercase` | 是否要求大写字母 | true | 密码管理 |
| `auth.password.requireLowercase` | 是否要求小写字母 | true | 密码管理 |
| `auth.password.requireDigit` | 是否要求数字 | true | 密码管理 |
| `auth.password.requireSpecialChar` | 是否要求特殊字符 | false | 密码管理 |
| `auth.password.historyCount` | 密码历史记录次数 | 1 | 密码管理 |
| `auth.password.maxAgeDays` | 密码有效期天数（0=永不过期） | 90 天 | 密码管理 |

## 4. 与其他模块的关系

| 关联模块 | 关系说明 |
| --- | --- |
| 用户管理 | 登录时查询用户信息、验证用户状态；停用/注销用户时调用会话失效能力 |
| 权限管理 | 登录成功后加载用户权限集 |
| 菜单与导航管理 | 登录成功后根据权限加载门户菜单和应用入口展示状态 |
| 微前端接入 | SSO Token 与登出事件传递给子应用 |
| 门户首页与个人中心 | 个人中心承载密码修改页面入口 |
| 系统管理 | 认证相关配置项由系统参数管理模块统一管理 |
