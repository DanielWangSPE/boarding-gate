# 05 - 权限管理

## 1. 模块概述

权限管理模块建设统一的权限模型和管控能力，覆盖功能权限（菜单、按钮）和数据权限（行级、列级）两个维度。后端接口鉴权复用功能权限标识，不单独维护 API 权限。作为统一门户平台的核心能力，本模块需要支持跨子应用的权限管控和权限聚合。

## 2. 权限模型

采用 **RBAC（基于角色的访问控制）** 为主、**ABAC（基于属性的访问控制）** 为辅的混合权限模型：

```
用户 --[分配]--> 角色 --[关联]--> 权限集
                                    ├── 功能权限（菜单、按钮）
                                    └── 数据权限（行级范围、列级字段）
```

> 后端 API 接口通过功能权限中的按钮权限标识统一鉴权（如 `@PreAuthorize("project:create")`），前后端共用同一套权限标识，无需单独管理 API 权限。

**权限合并策略**：

- 用户的最终权限 = 所有角色权限的并集
- 当存在明确的"拒绝"策略时，拒绝优先于允许

## 4. 业务规则汇总

| 编号 | 规则 | 说明 | 所属模块 |
| --- | --- | --- | --- |
| BR-01 | 权限并集 | 用户最终权限 = 所有角色权限的并集 | 功能权限 |
| BR-02 | 拒绝优先 | 存在明确拒绝策略时，拒绝优先于允许 | 功能权限 |
| BR-03 | 超管豁免 | 超级管理员拥有所有权限，不受限制 | 功能权限 |
| BR-04 | 前后一致 | 前端按钮权限标识与后端接口鉴权标识为同一个 | 功能权限 |
| BR-05 | 权限缓存 | 权限数据可缓存，角色权限变更时主动刷新缓存 | 权限查询与检验 |
| BR-06 | 子应用隔离 | 各子应用的权限资源相互隔离，但可在角色中聚合分配 | 功能权限 |
