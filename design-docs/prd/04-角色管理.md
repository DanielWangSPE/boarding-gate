# 04 - 角色管理

## 1. 模块概述

角色管理模块负责定义和维护系统中的角色信息，并提供角色权限配置和角色维度的用户分配能力。角色是连接用户和权限的核心纽带，通过将权限赋予角色、再将角色分配给用户，实现基于角色的权限管控。

## 3. 业务规则汇总

| 编号 | 规则 | 说明 | 所属模块 |
| --- | --- | --- | --- |
| BR-01 | 角色标识唯一 | 角色标识全局唯一，创建后不可修改 | 角色基本管理 |
| BR-02 | 内置角色保护 | 超级管理员等内置角色不可删除、不可修改角色标识 | 角色基本管理 |
| BR-03 | 删除前置检查 | 角色被用户引用时不可直接删除，需先解除所有关联 | 角色基本管理 |
| BR-04 | 权限实时生效 | 角色权限变更后，关联用户权限在下次请求时即时刷新 | 角色权限配置 |
| BR-05 | 停用即失效 | 角色停用后，关联用户对应权限立即失效 | 角色基本管理 |

## 4. 与其他模块的关系

| 关联模块       | 关系说明                                       |
| -------------- | ---------------------------------------------- |
| 用户管理       | 角色分配给用户                                 |
| 权限管理       | 角色关联权限集                                 |
| 菜单与导航管理 | 角色关联门户菜单访问权限，并影响应用入口可见性 |
| 组织架构管理   | 角色数据权限依赖组织架构                       |
| 微前端接入     | 子应用可注册自己的角色                         |
