# 06 - 菜单与导航管理

## 1. 模块概述

菜单与导航管理模块负责统一管理门户平台的菜单展示、导航结构和应用入口呈现。门户自身路由保持固定，子应用接入规则由微前端接入机制维护；本模块主要负责门户菜单、应用入口展示、权限过滤以及导航联动体验。

## 3. 业务规则汇总

| 编号 | 规则 | 说明 | 所属模块 |
| --- | --- | --- | --- |
| BR-01 | 菜单层级 | 菜单最多支持 4 级层级（目录 > 目录 > 菜单 > 按钮） | 菜单管理 |
| BR-02 | 权限过滤 | 无权限菜单不出现在用户界面和菜单 API 返回数据中 | 动态菜单 |
| BR-03 | 空目录隐藏 | 当目录下无任何可见菜单时，目录本身不展示 | 动态菜单 |
| BR-04 | 固定路由联动 | 门户固定路由通过守卫与导航状态联动，不承担子应用接入规则维护 | 路由联动 |
| BR-05 | 外链处理 | 外链类型菜单在新浏览器标签页中打开 | 菜单管理 |
| BR-06 | 删除限制 | 存在子菜单的菜单不可直接删除 | 菜单管理 |
| BR-07 | 应用入口展示 | 已接入子应用入口按权限展示，不随登录一次性加载子应用内部页面结构 | 动态菜单 |
| BR-08 | 应用入口过滤 | 子应用入口仅展示用户至少拥有一个菜单权限的应用 | 动态菜单 |
| BR-09 | 菜单缓存 | 已加载的门户菜单与导航状态可在会话内缓存，减少重复请求 | 动态菜单 |

## 4. 与其他模块的关系

| 关联模块 | 关系说明 |
| --- | --- |
| 权限管理 | 菜单关联权限标识，用于权限过滤 |
| 角色管理 | 通过角色分配菜单权限 |
| 微前端接入 | 子应用入口和接入规则由微前端接入模块维护，本模块负责其在门户中的展示与导航联动 |
| 认证与会话管理 | 门户固定路由守卫依赖登录态；会话失效、统一登出或登录态刷新后需同步更新菜单与导航状态 |
| 门户首页与个人中心 | 提供登录后的统一首页路由及个人中心相关入口，当前阶段首页仅承载欢迎页 |
